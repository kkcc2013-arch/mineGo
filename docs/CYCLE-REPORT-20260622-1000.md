- **类别**：性能优化（类别轮转：289 % 18 = 1）
- **优先级**：P1
- **状态**：new
- **涉及服务**：gym-service、social-service、backend/shared/websocket

### 需求内容
针对当前 WebSocket 实现存在的连接管理分散、消息发送效率低、内存占用高、心跳开销大等问题，提出：

1. **统一连接池管理器**：集中管理所有 WebSocket 连接，支持 10000+ 并发
2. **消息批处理器**：10ms 窗口内消息合并发送，减少网络往返
3. **心跳优化器**：分批次心跳检测，降低 CPU 开销 80%
4. **消息压缩中间件**：大消息自动压缩，减少带宽占用

### 预期收益
- 延迟降低 40-60%（P95）
- 吞吐量提升 200-300%
- 心跳峰值 CPU 降低 80%
- 内存占用减少 30%
