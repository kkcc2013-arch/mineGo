## Step 2: 实现 REQ-00193

### 需求信息
- **编号**：REQ-00193
- **类别**：技术债/重构
- **优先级**：P1
- **标题**：消除 console.log 与统一结构化日志使用

### 实现内容

#### gym-service 日志替换 (3处)
```javascript
// Before
console.log(`[Raid WS] User ${userId} joined raid ${raidId}`);

// After
logger.info({
  event: 'RAID_WS_JOIN',
  userId,
  raidId,
  participants: raidRooms.get(raidId).size
}, 'User joined raid WebSocket');
```

#### location-service 日志替换 (5处)
- Spawn 错误日志 → `logger.error({ err, spawnPointId, coords }, 'Spawn point processing error')`
- Spawn 完成日志 → `logger.info({ event: 'SPAWN_CYCLE_COMPLETE', spawnedCount }, ...)`
- GPS 欺骗检测 → `logger.warn({ event: 'GPS_SPOOF_DETECTED', userId, speedKmh }, ...)`
- Redis 缓存降级 → `logger.warn({ err, operation, fallback: 'database' }, 'Redis cache fallback')`

#### user-service 日志替换 (7处)
- 服务启动日志 → `logger.info({ event: 'SERVICE_READY', port, pid }, ...)`
- 服务启动失败 → `logger.error({ err, event: 'SERVICE_START_FAILED' }, ...)`
- SMS 验证码发送 → `logger.debug({ event: 'SMS_CODE_SENT', phone: phone.slice(-4) }, ...)`
- 家长同意邮件失败 → `logger.error({ err, userId, parentEmail }, ...)`
- 时区验证错误 → `logger.error({ err, timezone }, 'Timezone validation error')`
- 状态API错误 → `logger.error({ err, userId }, 'Get state/checksum/stats error')`

#### 新增检查脚本
- `backend/scripts/check-logging-consistency.js`
- `npm run check-logging` 命令
