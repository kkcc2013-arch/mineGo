## 📌 步骤 1: 生成新需求

### REQ-00584: API 超时策略标准化与分级超时治理系统

- **编号**: REQ-00584
- **类别**: API 设计规范 (索引 8)
- **优先级**: P1
- **状态**: new
- **涉及服务**: gateway, backend/shared/timeoutPolicy.js

### 背景
当前各服务超时配置分散在 circuitBreakers.js、EventBus.js 等文件中，缺乏统一治理：
- 硬编码超时值（15s/20s/30s/60s）没有按接口粒度区分
- 缺乏客户端可见的超时策略声明
- 无超时分级体系
- 缺少超时熔断联动机制

### 方案
- 四级超时体系（L1快速读/L2标准写/L3批量操作/L4流式长连接）
- TimeoutPolicyManager 集中式配置管理器
- 网关超时中间件（per-route enforcement）
- 客户端超时协商协议（X-Client-Timeout / X-Server-Timeout）
- 超时热更新 API（admin 端点）
- Prometheus 超时指标集成
