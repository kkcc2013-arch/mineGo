## 需求描述

为了提升游戏在低端网络环境下的启动速度和运行流畅度，需要实现一套基于用户行为预测和上下文感知（Context-Aware）的资源动态加载调度系统。该系统需根据玩家当前的地理位置、战斗需求、网络带宽状况，动态决定资源预加载的优先级。

## 技术方案

### 1. 行为预测引擎
- 基于用户历史交互数据，利用 LRU 变体算法预测玩家即将进入的区域或交互对象。
- 资源优先级排序：当前核心区域 > 预测区域 > 周边区域。

### 2. 加载器适配
- `resource-manager` 需提供异步加载 API，并支持基于 Service Worker 的后台预抓取。
- 提供 `ResourcePriority` 枚举：Critical, High, Medium, Low。

## 验收标准

- [ ] 核心资产在 3G 环境下加载延迟降低 30%。
- [ ] 预测区域资源加载正确率达到 80% 以上。
- [ ] 切换区域时，未加载资源不影响核心战斗功能。

## 影响范围

- `game-client/src/storage/`
- `game-client/src/api/`
