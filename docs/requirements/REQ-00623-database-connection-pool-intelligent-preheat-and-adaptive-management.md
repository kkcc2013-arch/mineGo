## 需求描述

当前数据库连接池配置是静态的，无法应对突发的流量波峰。在高峰期到来时，连接池容易出现饥饿现象，导致请求延迟。本需求旨在实现一个智能预热系统，根据历史负载趋势自动预热连接池，并根据实时连接使用率动态调整连接池大小，以减少连接建立开销。

## 技术方案

### 1. 流量趋势分析模块
- 基于现有性能指标(prometheus)，分析业务高峰的时间分布(cron-like schedule)。
- 实现一个周期性 job (`peakHourPreheater.js`)，在高峰前 5-10 分钟自动预热连接池。

### 2. 动态调整引擎
- 监控实时连接池使用率，如果连续 X 分钟高于 Y% (扩容) 或低于 Z% (缩容)。
- 使用 `ioredis` / `pg-pool` 的动态更新配置接口进行调整。
- 引入安全保护阈值（最小/最大连接数）。

## 验收标准

- [ ] 流量高峰前连接池能够自动预热到目标连接数
- [ ] 动态扩缩容逻辑不影响现有正常请求处理
- [ ] 系统具有最大连接数限额，防止内存资源被过度占用
- [ ] 提供相关监控指标可视化仪表盘
