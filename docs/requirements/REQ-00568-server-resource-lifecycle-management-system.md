## 需求描述

为了优化云基础设施的资源利用率，减少空闲或低效资源的成本开销，需要构建一套全自动的资源生命周期管理系统。该系统应能识别游戏集群内处于僵尸状态、长期无活跃连接、或超期未使用的计算和存储资源，并触发自动回收或休眠策略，从而降低不必要的运维成本。

## 技术方案

### 1. 资源状态监控与识别
- 集成资源监控指标（如 CPU 使用率、活跃连接数、内存占用），设定资源生命周期过期判定规则。
- 使用 K8s CRD (Custom Resource Definition) 标记资源所有者和最后使用时间。

### 2. 自动回收/休眠控制器
- 实现一个常驻控制器，根据预设的资源空闲阈值策略，对符合条件的资源自动执行：
    - 短期空闲：触发服务水平降级（扩缩容至最小）。
    - 长期空闲：触发备份后销毁（Pod 缩容、PVC 清理、云实例关机）。

### 3. 数据安全与防误删机制
- 执行资源销毁前，触发自动快照备份，备份保留 7 天后再清理。
- 引入人工二次确认机制，对于高价值资源禁止自动清理。

## 验收标准

- [ ] 实现监控资源活跃状态并标记不活跃实例。
- [ ] 系统能够对低活跃度 Pod 执行自动扩缩容至最小实例数。
- [ ] 长期闲置资源在触发回收前完成快照备份。
- [ ] 误删资源能够从备份中实现快速恢复。

## 影响范围

- K8s 集群管理服务
- 备份存储模块
