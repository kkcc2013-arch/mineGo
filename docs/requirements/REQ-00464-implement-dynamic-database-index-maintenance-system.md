## 需求描述

目前项目中存在大量静态索引，随着业务数据的增长，部分索引可能不再被高效利用，或者在某些查询模式下反而降低写入性能。需要构建一套动态数据库索引维护系统，监控索引使用情况，自动识别并清理无效/低效索引，并提供热加载索引创建策略。

## 技术方案

### 1. 索引使用监控模块
- 在 MySQL 层面利用 `sys.schema_unused_indexes` 监控无用索引。
- 收集慢查询日志中频繁出现的未索引字段，作为自动建议索引的依据。

### 2. 自动化维护流程
- 定时任务（`IndexMaintenanceJob`）每周分析索引利用率。
- 针对使用率低于阈值的索引，生成移除建议并发送给管理员审批（或在非高峰期自动删除）。

### 3. 动态配置更新
- 通过配置中心实现索引策略的动态调整，无需重启数据库服务。

## 验收标准

- [ ] 实现监控无用索引的 CronJob。
- [ ] 索引分析报告生成与自动通知。
- [ ] 索引创建与删除的标准化运维 API。
- [ ] 性能提升对比测试（删除冗余索引后，写入性能提升至少 10%）。
