## 需求描述

当前 mineGo 项目中虽然部署了分布式追踪系统 (Jaeger/OpenTelemetry)，但遇到接口延迟升高或错误率上涨时，仍依赖开发人员手动逐个排查链路。需求目标是构建一个自动根因分析引擎，通过分析链路时序数据、依赖拓扑及指标异常，自动定位故障发生的根本节点和异常类型。

## 技术方案

### 1. 异常检测层
- 集成 Prometheus 指标与 Trace 数据的相关性分析
- 实现基于阈值和波动检测的异常识别算法

### 2. 根因分析引擎
- 构建服务调用拓扑动态图
- 实现链路深度遍历算法，识别异常传播路径
- 引入启发式规则和统计分析，定位首个异常发生的节点

### 3. 可视化集成
- 将根因结果直接标注在追踪 UI 上
- 发送带有根因分析摘要的告警推送

## 验收标准

- [ ] 能够自动分析并高亮链路异常的根源节点
- [ ] 告警通知中包含简要的根因推断描述
- [ ] 能够准确识别至少 80% 的经典延迟/错误类故障
- [ ] 系统对业务链路的观测延迟影响在 5% 以内
