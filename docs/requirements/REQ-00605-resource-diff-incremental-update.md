## 需求描述

为了降低玩家在游戏更新时的流量消耗并提升更新速度，需要开发一套资源包差分更新与增量同步系统。该系统将对比当前版本与目标版本的资源包差异，仅下发差异部分的二进制补丁，从而实现快速更新。

## 技术方案

### 1. 差异计算 (Diffing)
- 服务器在每次资源版本发布时，通过 `bsdiff` 或自定义的资源对比算法生成补丁包。
- 使用哈希校验（SHA-256）确保差分包的完整性。

### 2. 客户端补丁应用 (Patching)
- 客户端在后台静默下载差分包。
- 使用 `bspatch` 库在本地重建完整资源文件。
- 如果补丁应用失败，系统自动回退至全量资源下载。

### 3. CDN 分发
- 优化 CDN 存储，将差分包与全量包分开存储。
- 客户端优先从 CDN 拉取最近版本的差分包，若版本跨度过大则采取分段式更新或全量拉取。

## 验收标准

- [ ] 差分包生成机制运行稳定，补丁体积较全量包减少至少 70%
- [ ] 客户端支持自动检测版本差异并应用补丁
- [ ] 应用补丁失败时具备平滑回退至全量下载的功能
- [ ] 全过程资源校验均通过 SHA-256 核对

## 影响范围

- `game-client`: 资源管理模块
- `resource-manager`: 资源打包服务
- `cdn-gateway`: 流量分发
