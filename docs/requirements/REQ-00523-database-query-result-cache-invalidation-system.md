## 需求描述

为了解决分布式环境下数据库数据更新导致缓存脏读问题，需构建一套基于数据库变更数据捕获 (CDC) 的智能缓存失效同步系统。该系统需自动监听底层数据库的操作（插入/更新/删除），并根据配置的规则自动清除或更新对应的缓存项，确保缓存的一致性，同时降低手动编写缓存清理逻辑的耦合。

## 技术方案

### 1. 变更数据捕获 (CDC) 适配器
- 使用 Debezium 或类似技术捕获 PostgreSQL/MySQL 的 Binlog/WAL。
- 过滤非必要的表和操作，仅针对核心业务表（如 Pokemon、Gym、Trade）进行监听。

### 2. 缓存失效同步引擎
- 编写 CDC 事件处理器，根据表名和主键映射到对应的缓存 Key 规则。
- 实现缓存失效广播机制，利用 Redis Pub/Sub 确保多实例环境下缓存清理的原子性。

### 3. 配置化管理
- 提供 YAML 配置，定义表与缓存 Key 的映射关系，支持正则表达式匹配。
- 增加防震荡保护机制（Debounce），避免高频变更导致的缓存穿透。

## 验收标准

- [ ] 数据库变更时，缓存能在 50ms 内实现失效或更新。
- [ ] 支持多实例分布式环境下的一致性保证。
- [ ] 提供配置接口，实现无需修改代码即可定义新的缓存失效规则。
- [ ] 具备完善的监控日志，记录缓存清除成功率和延迟。
