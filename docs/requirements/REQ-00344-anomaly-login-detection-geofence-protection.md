# REQ-00344：玩家账号异常登录检测与地理围栏防护系统

- **编号**：REQ-00344
- **类别**：安全加固
- **优先级**：P1
- **状态**：new
- **涉及服务/模块**：user-service、gateway、backend/shared、Redis、PostgreSQL、admin-dashboard
- **创建时间**：2026-06-27 02:00 UTC
- **依赖需求**：REQ-00219（会话异常检测与自动防护系统）

## 1. 背景与问题

当前系统已实现会话异常检测（REQ-00219），但缺少基于地理位置的异常登录检测机制。现实场景中，玩家账号可能遭遇以下安全威胁：

1. **异地登录攻击**：攻击者窃取凭证后从不同地理位置登录，当前系统无法及时识别
2. **地理围栏缺失**：无法限制账号只能在特定地理范围内使用，导致账号被盗用风险
3. **异常登录模式识别不足**：缺少对短时间内跨地域登录、异常时段登录等行为的检测
4. **响应机制滞后**：发现异常后缺少自动防护措施（如临时锁定、二次验证）

根据 STATUS.md，安全维度已达满分，但地理围栏和精细化异常检测仍有提升空间。

## 2. 目标

建立多层账号安全防护体系，实现：

1. **地理位置异常检测**：识别短时间内跨城市/国家的异常登录行为
2. **地理围栏机制**：支持玩家设置常用地理位置，超出范围自动触发安全验证
3. **智能风控评分**：基于多维度因子（位置、设备、时间、行为模式）计算风险分值
4. **自动化响应**：高风险登录自动触发 MFA 验证、临时锁定或管理员告警

目标：将账号被盗用后的损失降低 90%，异常登录检测准确率达 95%+。

## 3. 范围

- **包含**：
  - 登录地理位置采集与存储（基于 GPS 或 IP 定位）
  - 地理围栏配置接口（玩家设置常用城市/区域）
  - 异常登录模式识别算法（速度检测、跨地域检测）
  - 风险评分引擎（位置+设备+时间+历史行为）
  - 自动化响应机制（MFA 触发、临时锁定、告警通知）
  - 管理后台可视化监控面板

- **不包含**：
  - 支付风控系统（已有独立需求）
  - 实时位置追踪（仅登录时采集）
  - 第三方风控服务集成（本迭代不涉及）

## 4. 详细需求

### 4.1 登录地理位置采集

```javascript
// 登录时采集位置数据
{
  userId: String,
  loginTime: Date,
  location: {
    type: 'gps' | 'ip',
    coordinates: [longitude, latitude],
    city: String,
    country: String,
    accuracy: Number // 米
  },
  device: {
    deviceId: String,
    deviceType: String,
    os: String,
    browser: String
  },
  ip: String
}
```

- 支持 GPS 精确定位（移动端）和 IP 定位（Web 端）
- 使用 PostGIS 存储 Point 类型地理位置
- 登录记录保留 180 天用于历史分析

### 4.2 地理围栏配置

```sql
CREATE TABLE user_geofences (
  id SERIAL PRIMARY KEY,
  user_id VARCHAR(64) NOT NULL,
  name VARCHAR(100),
  center GEOGRAPHY(POINT, 4326),
  radius_meters INTEGER DEFAULT 50000, -- 默认 50km
  is_primary BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

- 每个用户最多配置 5 个常用地理围栏
- 支持设置主要围栏（严格模式）和次要围栏（宽松模式）
- 围栏外登录触发二次验证

### 4.3 异常登录检测算法

**速度检测**：计算两次连续登录的地理位置距离和时间差
```javascript
// 速度 = 距离 / 时间间隔
const speed = distance / timeDiffHours; // km/h
if (speed > 800) { // 超过飞机速度
  riskScore += 40;
}
```

**跨地域检测**：
- 同一账号 1 小时内跨城市登录 → 风险 +30
- 同一账号 24 小时内跨国家登录 → 风险 +50

**异常时段检测**：
- 用户历史登录时段分布建模
- 非常规时段（如凌晨 3-5 点）首次登录 → 风险 +20

**设备异常检测**：
- 新设备 + 新位置 → 风险 +25
- 已知设备 + 新位置 → 风险 +10

### 4.4 风险评分引擎

```javascript
// 风险等级定义
const RISK_LEVELS = {
  LOW: { min: 0, max: 30, action: 'allow' },
  MEDIUM: { min: 31, max: 60, action: 'challenge' }, // MFA
  HIGH: { min: 61, max: 80, action: 'lock_temp' }, // 临时锁定
  CRITICAL: { min: 81, max: 100, action: 'lock_permanent' } // 永久锁定
};
```

多维度权重配置：
- 地理位置：40%
- 设备指纹：25%
- 时间模式：20%
- 历史行为：15%

### 4.5 自动化响应机制

| 风险等级 | 自动响应 | 通知 |
|---------|---------|------|
| MEDIUM (31-60) | 强制 MFA 验证 | 邮件/推送通知 |
| HIGH (61-80) | 临时锁定 30 分钟 + MFA | 邮件/推送/短信 |
| CRITICAL (81-100) | 永久锁定 + 人工审核 | 邮件/推送/短信 + 管理员告警 |

### 4.6 API 接口设计

```yaml

```
