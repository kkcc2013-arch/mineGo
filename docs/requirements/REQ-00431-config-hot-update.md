## 需求描述

当前 mineGo 项目中，服务配置多采用静态加载，变更需要重启服务。为提升运维效率与系统稳定性，需求建立一套分布式配置中心，支持：
1. 动态配置发布：支持不停机更新业务参数（如熔断阈值、限流配额、开关控制）。
2. 版本控制与回滚：所有配置变更均有版本号，支持一键快速回滚。
3. 灰度配置发布：支持配置按服务实例分组进行灰度下发。
4. 配置一致性保证：确保在分布式环境下配置推送的最终一致性。

## 技术方案

### 1. Config Service 核心模块
- 基于 Etcd 或 Zookeeper 构建强一致性配置存储。
- 提供 gRPC/HTTP API 供服务订阅配置变更通知（Watch 机制）。

### 2. 客户端集成 (shared/config)
- 实现 Config SDK，服务启动时加载基础配置，并建立长连接监听配置变更。
- 配置变更触发回调机制，允许各模块自定义热更新逻辑（如重新加载 Redis 连接池大小）。

### 3. 后台管理 (admin-dashboard)
- 提供可视化配置编辑界面，集成 Git 存储后端实现配置历史版本追溯。
- 实现发布审批流与预览机制。

## 验收标准

- [ ] 实现配置服务中心，支持配置存取与监听。
- [ ] 客户端集成 SDK，实现配置变更秒级热更新。
- [ ] 配置修改支持版本记录，并在管理后台提供一键回滚功能。
- [ ] 支持灰度配置发布，仅在指定环境/实例集群生效。
- [ ] 系统具备配置变更审计日志。
