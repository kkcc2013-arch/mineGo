## 需求描述

为了降低数据库压力并提升接口响应速度，引入 Redis 作为分布式缓存层。核心逻辑包括：热点数据读取缓存、更新数据同步失效缓存、以及过期策略配置。

## 技术方案

### 1. 缓存客户端封装
- 使用 `go-redis` 库进行封装，实现统一的 `CacheProvider` 接口。
- 支持简单的 Get/Set/Delete 操作。

### 2. 缓存策略实现
- 在 `core-service` 中实现 Cache-Aside 模式。
- 对于高频查询接口，优先从 Redis 读取。

## 验收标准

- [ ] Redis 缓存服务已成功集成并运行
- [ ] 缓存命中率达到 70% 以上
- [ ] 数据更新时，相应的缓存项能正确失效
- [ ] 缓存故障时，系统能降级直接读库
