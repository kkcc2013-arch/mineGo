## 需求描述

为了实时掌握系统的健康状况，我们需要引入一种基于滑动窗口的智能告警系统。该系统应能够自动计算 API 请求的延迟基准（Baseline），并在请求延迟显著偏离基准时自动触发告警，支持减少误报。

## 技术方案

### 1. 基准计算模块
- 基于 Prometheus 采集的延迟直方图指标。
- 使用滑动窗口（如：最近 1 小时）计算 P95/P99 延迟的移动平均值和标准差。
- 动态调整阈值（Mean + 3 * StdDev）。

### 2. 告警判决引擎
- 如果连续 M 个数据点偏离阈值，触发告警。
- 通过 Alertmanager 发送告警通知至监控看板或 Slack/钉钉。

### 3. 可视化
- 在 Grafana 中创建延迟基准对比图，展示实时延迟与动态阈值的对比。

## 验收标准

- [ ] 实现延迟基准计算算法。
- [ ] 针对延迟异常实现自动告警。
- [ ] 告警能够显示受影响的 API 端点。
- [ ] 在监控仪表盘中直观展示异常状态。
