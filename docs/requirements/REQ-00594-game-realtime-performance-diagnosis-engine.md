## 需求描述

开发一个能够实时监测游戏客户端和后端微服务之间交互性能的诊断引擎。该引擎能够自动识别由于网络延迟、后端响应缓慢或资源瓶颈导致的丢帧或操作阻塞问题，并实时推送到运维监控仪表板。

## 技术方案

### 1. 客户端监测 SDK
- 在游戏渲染循环中插入性能埋点，记录帧耗时 (Frame Time) 和 API 延迟 (Latency)。
- 当触发阈值（如连续 3 帧耗时超过 50ms）时，采集当前的 CPU/内存状态。

### 2. 后端关联分析
- 使用 OpenTelemetry 的 `trace_id` 追踪客户端性能事件到对应的微服务调用链路。
- 自动分析该链路中耗时最长的服务环节。

## 验收标准

- [ ] 客户端在性能卡顿时能自动上报诊断数据。
- [ ] 监控平台能关联并显示该时刻的服务调用链。
- [ ] 系统能自动区分是客户端渲染瓶颈还是后端接口慢。

## 影响范围
