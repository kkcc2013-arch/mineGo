## 需求描述

随着用户活跃度增加，现有的 WebSocket 连接架构在处理高频消息推送时存在连接耗尽、Redis 压力过大及消息发送频繁触发 TCP 拥塞的问题。本需求旨在构建一套连接池与消息批处理系统，提升系统吞吐量。

## 技术方案

### 1. WebSocket 连接池与路由优化
- 实现基于 Redis 的分布式 WebSocket 连接管理，支持连接的跨节点状态共享。
- 引入连接限流器，防止单点故障引发的连接风暴。

### 2. 消息批处理引擎
- 设计消息缓冲队列（Buffer Queue），支持将短时间内产生的多条小消息合并为一批次发送。
- 引入发送窗口控制（Send Window），通过动态调整窗口大小适应不同网络环境下的吞吐量。

### 3. 代码实现逻辑
- 使用异步 I/O (asyncio) 优化连接处理逻辑。
- 在 `backend/shared/websocket` 模块中增加 `BatchSender` 类。

## 验收标准

- [ ] WebSocket 连接支持跨 Redis 实例的状态查询。
- [ ] 消息批处理开启后，吞吐量提升至少 30%。
- [ ] 系统在高负载下仍能保持低延迟发送消息。
- [ ] 实现针对异常连接的自动断开与清理机制。

## 影响范围

- gateway (网关层)
- backend/shared/websocket (核心通信库)
- Redis (集群部署)
