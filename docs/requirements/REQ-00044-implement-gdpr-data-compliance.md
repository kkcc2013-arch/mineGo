## 需求描述

为了符合 GDPR (通用数据保护条例) 要求，系统需提供用户自主下载个人数据以及彻底删除个人数据的能力。目前系统缺少对用户个人数据的导出（Portable Data）以及完全擦除（Right to be Forgotten）的支持。

## 技术方案

### 1. 数据导出模块
- 实现异步导出任务，将用户 profile、捕捉记录、道馆记录聚合为 JSON 格式文件。
- 将文件上传至 S3 临时存储，并生成过期时间为 24 小时的签名 URL 通知用户。

### 2. 数据擦除模块
- 实现“软删除”到“硬删除”的转换，确保在 30 天宽限期后彻底从生产库及备份中清理用户数据。
- 引入事件驱动机制，确保用户删除指令同步到下游子系统（如社交服务、支付服务）。

## 验收标准

- [ ] 用户可以通过前端 UI 申请导出个人所有历史数据。
- [ ] 导出的 JSON 数据结构完整且包含必要字段。
- [ ] 用户可以申请账号删除，系统进入删除冷却期并向用户确认。
- [ ] 冷却期满后，用户数据在所有关联服务中被自动清理。

## 影响范围

- `user-service`: 账号删除逻辑、数据查询服务
- `data-service`: 聚合导出服务
- `api-gateway`: 用户设置接口扩充
