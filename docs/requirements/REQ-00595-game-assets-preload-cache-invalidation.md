## 需求描述

针对游戏资源（图片、3D模型、音效）加载缓慢的问题，设计一套智能预加载方案。当用户在地图某个区域时，根据用户的移动轨迹概率，预先加载相邻区域资源。同时，引入版本哈希校验，实现更精准的缓存失效逻辑。

## 技术方案

### 1. 资源预加载引擎
- 在 `game-client` 中实现资源池管理器。
- 基于用户坐标与轨迹，动态计算资源请求队列，优先级排序。

### 2. 缓存失效策略
- 基于 `content-hash` 的缓存文件名。
- 使用 `cdn-gateway` 的热更新接口即时触发缓存失效。

## 验收标准

- [ ] 实现资源池优先队列管理逻辑。
- [ ] 验证哈希版本更新后，客户端能立即获取新资源。
- [ ] 监控预加载带来的流量与加载速度优化指标。

## 影响范围

- game-client/assets/loader.js
- cdn-gateway/config/caching.yaml
