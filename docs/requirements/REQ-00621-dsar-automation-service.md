## 需求描述

为了满足 GDPR 和 CCPA 等数据保护法规，用户应具备请求查询、修改或删除其个人数据的权利。目前人工处理此类 DSAR (Data Subject Access Request) 请求效率低下且易出错。本需求旨在构建一个自动化平台，支持从用户提交请求到自动生成数据导出报表、自动触发数据清理的一站式闭环流程。

## 技术方案

### 1. 请求接入层
- 在用户设置页面新增 "数据隐私与个人信息导出" 功能入口。
- 提供三种标准请求类型：数据导出 (Export)、数据更正 (Rectification)、数据删除 (Erasure)。
- 请求需通过双重身份验证 (MFA) 确认合法性。

### 2. 后端处理中心
- 构建 DSAR 处理工作流引擎，利用 task-queue 处理耗时的数据查询和备份任务。
- 各个关联服务（user-service, game-service, logs-processor）需实现数据脱敏和导出接口，遵循统一的标准 API 规范。
- 对于删除请求，实现 "软删除" -> "定时清理" -> "最终匿名化" 的生命周期管理。

### 3. 数据审计
- 所有 DSAR 请求需留痕，存入只读审计日志中，以备合规性审查。

## 验收标准

- [ ] 支持用户在客户端发起数据导出请求，并在 48 小时内自动生成安全下载链接。
- [ ] 支持用户请求删除其所有个人历史记录，且系统能在 7 天内完成合规性清理。
- [ ] 自动化流水线需包含敏感信息过滤机制，确保导出内容不包含未授权数据。
- [ ] 流程日志完整且不可篡改。

## 影响范围

- user-service: 处理个人信息定位
- data-platform: 负责跨服务的数据聚合与脱敏
- gateway: 负责请求身份鉴权
