## 需求描述

为了解决生产环境中偶发的 Node.js 服务内存占用持续上升的问题，需要构建一套自动化的内存分析和诊断工具链，支持定时触发 Heap Snapshot 采样，并自动分析泄露点。

## 技术方案

### 1. 自动快照触发器
- 集成 `heapdump` 模块
- 根据阈值（内存占用超过 80%）自动生成快照。

### 2. 内存分析分析引擎
- 使用 `llnode` 或 `heapsnapshot` 解析库对快照进行对比分析。
- 生成对象增长趋势报表。

### 验收标准

- [ ] 内存使用率超过 80% 时自动记录快照。
- [ ] 系统能够比对两次快照并识别泄露对象。
- [ ] 告警系统能通知运维团队内存溢出风险。

## 影响范围

- `backend-gateway`, `backend-services`
