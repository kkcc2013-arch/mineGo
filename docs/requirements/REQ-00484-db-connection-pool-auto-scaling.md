## 需求描述

为了解决数据库在流量波动时的连接池竞争、死锁及连接失效问题，需要构建一套智能化的数据库连接池管理系统。该系统需根据实时负载自动动态调整连接池大小，并对空闲连接进行主动健康巡检以防止陈旧连接问题。

## 技术方案

### 1. 动态负载感知模块
- 通过 Prometheus 指标实时获取数据库负载（QPS, 并发连接数）。
- 设计 PID 控制算法或基于规则的自动伸缩策略（Auto-scaler）。

### 2. 健康巡检与重连模块
- 实现定期对 idle 连接执行心跳检测 (SELECT 1)。
- 当检测到连接异常时，立即从池中剔除并优雅重连，避免请求阻塞。

### 3. 连接池溢出保护
- 当连接池达到最大限制时，引入排队机制与快速失败策略。

## 验收标准

- [ ] 实现连接池大小根据负载动态调整。
- [ ] 能够检测并清理无效的陈旧连接。
- [ ] 提供连接池健康度实时指标接口。
- [ ] 压测下连接池不会因耗尽导致服务中断。
