### 选择的需求

**标题**: 延迟任务队列与可靠重试机制  
**优先级**: P1  
**类别**: 技术债/重构  
**状态**: new → **done** ✅

### 实施完成

#### ✅ 核心模块实现

1. **DelayQueue.js** - 延迟队列核心类
   - 支持延迟任务调度（秒/分/小时/天级）
   - 实现指数退避重试机制（2^n + 抖动）
   - 支持任务优先级（critical/high/normal/low）
   - 死信队列自动处理和告警

2. **delayBucketScheduler.js** - 延迟桶调度器
   - 多级延迟桶（immediate/1m/5m/15m/1h/6h/24h）
   - 自动任务迁移
   - 高效调度算法

3. **delayQueueMonitor.js** - 队列监控服务
   - DLQ 监控和告警
   - 健康检查
   - 自动重试

#### ✅ 服务集成

1. **gym-service** 
   - 集成 raidRewardHandler
   - 实现 Raid 奖励延迟发放（5分钟）
   - 高优先级任务（maxRetries=10）

2. **payment-service**
   - 集成 orderTimeoutHandler
   - 实现订单超时自动取消（30分钟）
   - 关键优先级任务（maxRetries=3）

#### ✅ 管理 API

- **gateway** 已挂载 `/api/admin/delay-queue` 路由
- 提供 4 个管理端点：
  - GET /stats - 获取队列统计
  - GET /health - 获取健康状态
  - POST /tasks - 手动调度任务
  - POST /dlq/:taskId/retry - 重试 DLQ 任务
