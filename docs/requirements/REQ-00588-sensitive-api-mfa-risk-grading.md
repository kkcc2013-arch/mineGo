## 需求描述

针对涉及用户资产、隐私或敏感操作的 API，引入二次身份验证（如 MFA 或风控验证），并根据请求的风险级别进行自动分级拦截。目标是降低由于账号被盗或会话劫持导致的安全风险。

## 技术方案

### 1. 风险分级引擎
- 在 `gateway` 引入风险评估中间件，根据 IP 频率、地理位置、设备特征、会话时效等多维度指标计算实时风险分值。

### 2. 二次验证协议
- 敏感 API 响应触发 403 Forbidden 并携带验证要求（例如 MFA_REQUIRED_TOKEN）。
- 客户端接收后引导用户完成二次验证，获取短期有效的验证令牌。

### 3. API 鉴权增强
- 敏感 API 校验验证令牌的合法性与时效性。

## 验收标准

- [ ] 敏感 API 调用必须根据风险分级判定是否触发二次验证。
- [ ] 验证完成后获取的令牌支持跨服务鉴权。
- [ ] 风控系统能够有效拦截高风险行为调用。
- [ ] 相关的风控日志需加密存储并支持溯源。

## 影响范围

- `gateway`, `user-service`, `backend/security`
