## 需求描述

随着游戏图形复杂度增加，客户端渲染性能波动可能严重影响用户体验。本需求旨在建立一套客户端渲染性能指标监控系统，能够实时监测帧率(FPS)、绘制调用(Draw Calls)、显存占用(GPU Memory)等关键指标，并针对异常波动触发预警。

## 技术方案

### 1. 客户端性能埋点
- 集成性能数据采集 SDK，支持在客户端各生命周期自动采集帧率、绘制压力等数据。
- 采样频率根据电量状态动态调整，降低对设备性能影响。

### 2. 指标上报与聚合
- 采用批处理上报策略，通过高压缩比协议上报性能数据至监控系统。
- 后端服务对数据按设备型号、地理位置、App版本进行维度聚合。

### 3. 告警阈值计算
- 基于滑动窗口统计，自动建立各设备层级的性能基准线。
- 当 FPS 连续低于阈值或显存占用溢出时，触发告警并自动关联该时段的崩溃日志。

## 验收标准

- [ ] 客户端支持 FPS、Draw Calls、显存占用指标采集与上报。
- [ ] 后端支持按设备型号聚合渲染性能指标。
- [ ] 实现针对客户端渲染性能异常的自动化告警机制。

## 影响范围

- game-client (核心性能采集模块)
- performance-monitor (指标聚合与告警引擎)
