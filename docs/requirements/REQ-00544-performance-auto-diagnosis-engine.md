## 需求描述

为了解决日益复杂的分布式服务性能问题，需要建立一个自动诊断引擎，能够实时分析采集到的性能指标（如 API 响应时间、数据库查询耗时、队列积压情况等），并根据预设的性能模型和异常模式，自动定位性能瓶颈并提供优化建议。

## 技术方案

### 1. 性能指标采集器优化
- 增加对关键路径的分布式追踪（OpenTelemetry）的细粒度覆盖。
- 引入指标数据的实时聚合与异常流式计算（基于 Flink 或类似的流计算架构）。

### 2. 自动诊断引擎
- 定义性能异常模式库（如 N+1 查询、锁争用、线程阻塞、内存泄露模式）。
- 诊断算法：使用规则引擎和简单的启发式算法来匹配异常模式。

### 3. 诊断报告与告警
- 自动生成性能诊断报告，并推送到管理员告警系统（钉钉/Slack/邮件）。

## 验收标准

- [ ] 性能数据采集覆盖率达到 95% 以上。
- [ ] 诊断引擎能够识别至少 5 种预定义的性能异常模式。
- [ ] 诊断报告生成耗时控制在 5 分钟以内。
