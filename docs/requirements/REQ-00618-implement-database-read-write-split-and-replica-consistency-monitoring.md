## 需求描述

为了应对高并发业务场景，当前后端服务需要通过数据库读写分离来分担主库压力。然而，读写分离引入了副本延迟（Replica Lag）的风险，可能导致业务逻辑读取到旧数据。本项目需求在于实现一套透明的读写分离方案，并配套实时副本延迟监控，在检测到延迟超过阈值时，自动切换到强一致性读取模式（读主库）。

## 技术方案

### 1. 读写分离中间件
- 在 `database` 层引入轻量级路由逻辑。
- 通过配置项控制读库负载均衡策略（轮询/权重）。

### 2. 副本延迟监控
- 在 `backend/jobs` 中增加定时心跳任务，在主库写入时间戳，并测量副本库接收到的时间。
- 暴露 Prometheus 指标供集群监控告警使用。

### 3. 强一致性切换
- 在 `gateway` 层引入一致性标志位（`X-Consistency-Level`）。
- 服务层在处理特定敏感业务（如交易、账户变动）时，自动降级读请求至主库。

## 验收标准

- [ ] 实现读写路由切换，主从库自动分流。
- [ ] 实现副本延迟监控任务，延迟超过 500ms 时产生告警。
- [ ] 敏感业务能通过Header强制读主库。
- [ ] 性能测试验证读库压力降低 30% 以上。
