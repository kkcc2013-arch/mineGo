## 需求描述

为了更灵活地调整游戏玩法参数、开启/关闭特定功能以及开展 A/B 测试，需要构建一个中心化的动态配置与实验管理平台。该平台应支持在不发布新代码的情况下，通过管理后台动态修改配置，并针对不同用户群体（例如实验组 vs 对照组）应用不同的配置策略。

## 技术方案

### 1. 配置中心与分发服务
- 实现基于 Redis 的动态配置中心，支持热更新。
- 引入用户分层/分群引擎，计算用户在实验中的分组（基于 UserID 哈希）。

### 2. 管理后台
- 在 `admin-dashboard` 中开发配置编辑与实验设置 UI。
- 支持配置版本控制和发布审批。

### 3. 客户端适配
- `game-client` 定期拉取或接收配置中心推送，更新本地运行参数。

## 验收标准

- [ ] 实现动态配置管理后台
- [ ] 支持按用户 ID 进行 A/B 分组
- [ ] 验证配置修改后无需重启客户端生效
- [ ] 配置修改记录支持审计和回滚
