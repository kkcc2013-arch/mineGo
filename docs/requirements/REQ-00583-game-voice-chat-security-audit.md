## 需求描述

为了满足全球各地区的网络安全法规（如 COPPA, GDPR, 及各地区游戏运营合规性要求），需要建立一套自动化的游戏内实时语音聊天内容审计系统。该系统需在不显著影响用户通话体验的前提下，对语音内容进行实时转化为文本，识别不当言论（如暴力、色情、欺凌等），并采取自动化处置措施。

## 技术方案

### 1. 语音内容处理流
- 客户端集成 WebRTC 音频采集模块，将音频流实时推送到 `voice-chat-service`。
- `voice-chat-service` 调用 AI 语音转文本 (STT) 引擎进行实时转换。

### 2. 内容审核引擎
- 转换后的文本流推送到 `compliance-analyzer`，使用敏感词库和基于深度学习的语义分析模型进行风险评分。
- 对高风险言论自动触发禁言、警告或上报审计功能。

### 3. 系统集成
- 增加 WebSocket 消息转发链路，确保审计服务的高可用性和低延迟。
- 对审计结果记录日志，供后续运营审核查询。

## 验收标准

- [ ] 语音转文本系统成功部署并实现 < 300ms 的处理延迟。
- [ ] 语音审核系统能准确识别至少 90% 的预设敏感词。
- [ ] 针对高风险行为，能够自动触发用户禁言操作。
- [ ] 系统审计记录完整且包含时间戳和用户关联 ID。

## 影响范围

- `social-service`: 增加相关处理逻辑。
- `gateway`: 增加 WebSocket 路由配置。
- `infrastructure`: 引入新的语音审计 AI 模型微服务。
