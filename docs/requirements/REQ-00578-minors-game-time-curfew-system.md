## 需求描述

为了遵守未成年人保护法规，需要设计一套强制性的游戏时长限制和宵禁系统。该系统将根据用户身份信息中的年龄字段，对未成年用户实施每日游戏时长上限限制，以及在特定时间段内禁止登录的宵禁功能。

## 技术方案

### 1. 用户身份校验模块
- 在用户登录及更新用户信息时，同步检查年龄信息。
- 利用 Redis 存储未成年用户身份映射，以便快速查询。

### 2. 时长追踪与宵禁引擎
- 引入 Node.js 中间件，实时记录未成年用户的在线时长。
- 当达到时长上限或进入宵禁时间段，触发中断响应，强制用户下线并禁止再次登录。

### 3. 前端交互
- 在游戏客户端添加剩余时长提醒功能。
- 在宵禁时间段显示友好的提示信息。

## 验收标准

- [ ] 未成年用户在达到每日允许时长后，会被强制踢下线。
- [ ] 宵禁时间段内，未成年用户无法成功发起游戏请求。
- [ ] 限制功能不影响成年用户的使用。

## 影响范围

- user-service: 年龄字段处理逻辑
- gateway: 在线时长追踪中间件
- game-client: 状态处理与用户提示
