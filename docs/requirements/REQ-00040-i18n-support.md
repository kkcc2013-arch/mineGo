## 需求描述

为了支持全球用户，需要引入国际化支持，包括语言切换、日期格式化和货币格式化。

## 技术方案

### 1. 语言资源管理
- 引入 i18next 作为核心库。
- 将翻译字符串提取到单独的 JSON 文件中（en-US, zh-CN, ja-JP）。

### 2. 核心架构适配
- 接口层增加 `Accept-Language` 头处理。
- 后端错误信息根据用户语言环境返回对应文案。

## 验收标准

- [ ] 切换语言后 UI 全部翻译为对应语言。
- [ ] 日期和时间格式符合当地习惯。
- [ ] 后端报错信息支持多语言返回。

- core-api/middleware/i18n.go
