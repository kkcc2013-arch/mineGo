## 需求描述

目前游戏客户端资源更新采用全量包替换方式，导致移动端用户在小版本更新时消耗大量流量且更新速度慢。需要实现资源包的差异化增量更新系统，仅下发差异数据(Diff)，并支持客户端本地合并。

## 技术方案

### 1. 差异包生成与存储
- 服务端增加资源快照机制，基于哈希指纹对比，生成BSDiff或类似算法的增量补丁文件。
- 将补丁存储至CDN，并在资源清单(manifest)中加入增量包索引。

### 2. 客户端合并逻辑
- 客户端下载清单后，对比本地文件哈希，若存在对应增量包则下载。
- 在本地存储空间执行Patch操作，验证完整性后再替换原始文件。

## 验收标准

- [ ] 实现服务端资源快照与差异计算自动化脚本
- [ ] 客户端支持增量更新的下载与Patch合并流程
- [ ] 确保增量更新后的资源哈希校验通过
- [ ] 更新流程在网络波动环境下具备断点续传功能

## 影响范围

- asset-service (新增差分计算逻辑)
- game-client (新增Patch合并模块)
- cdn-gateway (资源分发策略更新)
