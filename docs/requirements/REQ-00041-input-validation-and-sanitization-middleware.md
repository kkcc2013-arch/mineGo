## 需求描述

为了防止恶意用户通过 API 注入攻击（如 SQL 注入、XSS 等），需要实现一套强制性的统一输入验证与清理中间件。所有进入系统的 HTTP 请求必须通过此中间件进行模式校验与敏感字符过滤。

## 技术方案

### 1. 验证中间件实现
- 使用 Zod 作为验证定义库，定义标准的请求对象 Schema。
- 在 API 网关层统一挂载验证中间件，确保请求参数类型、格式安全。

### 2. 清理逻辑
- 对所有 String 类型的输入执行 HTML 实体编码或删除敏感字符（基于白名单）。
- 记录校验失败请求并上报至安全监控服务。

## 验收标准

- [ ] 所有 API 接口配置了 Zod 校验 Schema。
- [ ] 恶意注入请求能被及时拦截并返回 400 错误码。
- [ ] 验证失败日志能够实时推送到日志聚合平台。

## 影响范围

- API 网关（请求预处理）
- 各业务服务（定义 Schema）
