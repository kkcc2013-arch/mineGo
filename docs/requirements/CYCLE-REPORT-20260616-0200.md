**状态**：done ✅

#### 实现概述

成功引入事件总线适配器抽象层，支持多种消息系统动态切换：

1. **IEventBusAdapter 接口**
   - 定义统一的适配器接口
   - 包含 connect、disconnect、publish、subscribe、healthCheck 方法
   - 提供基础指标统计功能

2. **KafkaAdapter**
   - 封装现有 kafkajs 逻辑
   - 支持 DLQ、重试、metrics
   - 与现有 EventBus 功能完全兼容

3. **MemoryAdapter**
   - 基于 EventEmitter 和内存队列
   - 支持异步消息投递、重试机制
   - 适用于开发、测试环境

4. **RedisStreamAdapter**
   - 基于 Redis Streams（XADD/XREAD/XGROUP）
   - 支持消费者组模式
   - 提供死信处理和超时重投递
   - 适用于轻量级部署

5. **EventBus 重构**
   - 使用适配器模式
   - 提供 `createEventBus` 工厂方法
   - 通过 `EVENT_BUS_ADAPTER` 环境变量动态选择
   - 保持 API 向后兼容

#### 环境变量支持
```bash
EVENT_BUS_ADAPTER=kafka    # 生产环境（默认）
EVENT_BUS_ADAPTER=memory   # 开发/测试
EVENT_BUS_ADAPTER=redis    # 轻量级部署
```
