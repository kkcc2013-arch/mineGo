## 需求描述

为了解决日益复杂的接口响应时间波动问题，需要建立一套全链路性能分析框架，能够自动定位慢查询、锁竞争以及不必要的 I/O 操作。

## 技术方案

### 1. 性能采样中间件
- 在 Gateway 层添加性能采样拦截器，对 5% 的请求进行全链路标记。
- 记录请求处理过程中各阶段耗时（数据库、缓存、外部 API）。

### 2. 分析引擎
- 在 `backend/shared/perf` 中集成采样聚合逻辑。
- 自动生成性能热点分析报告，通过 Prometheus 指标暴露。

## 验收标准

- [ ] 性能采样中间件已上线，采样比例可配置。
- [ ] 性能热点分析报告每日自动推送。
- [ ] 关键业务接口耗时统计指标已接入 Prometheus。

## 影响范围

- gateway, 所有后端服务
