### 2. 实现需求

**REQ-00092: API 请求合并与批量查询优化**

| 字段 | 值 |
|------|-----|
| 状态 | done |
| 工作量 | M（中等） |
| 实际耗时 | ~3 小时 |

#### 前端实现

**BatchApiClient** (`frontend/game-client/src/api/BatchApiClient.js`)
- 请求去重：100ms 窗口期内相同请求自动合并
- 批量查询：`batchGetPokemonDetails()`, `batchGetSpecies()`, `batchGetFriendStatus()`, `batchGetGyms()`
- 智能合并：50ms 窗口期内请求自动排队合并
- 内存缓存：LRU 策略，最大 200 条，TTL 5 分钟

#### 后端实现

**pokemon-service**
- `POST /batch/details` - 批量精灵详情（最多 50 个 ID）
- `POST /batch/species` - 批量种族数据（最多 100 个 ID）
- `POST /batch/iv` - 批量 IV 计算（最多 100 条）
- Redis 缓存 + Prometheus 指标

**social-service**
- `POST /batch/friends/status` - 批量好友状态（最多 100 个）
- `POST /batch/friends/summary` - 精简版好友摘要（最多 200 个）
- 在线状态 Redis 查询 + Prometheus 指标

**gym-service**
- `POST /batch/details` - 批量道馆详情（最多 50 个）
- `POST /batch/nearby` - 批量附近道馆（PostGIS 空间查询）
- `POST /batch/raids` - 批量 Raid 信息
- Redis 缓存 + Prometheus 指标

#### 性能预期
- 网络请求减少 70%+
- 页面加载速度提升 40%+
- 精灵收藏页（30 个）从 30 个请求 → 1 个批量请求
