## 需求描述

为了提升游戏体验，客户端缓存了大量动态数据（如精灵列表、道馆信息、玩家配置）。当前的缓存更新逻辑依赖后端主动推送或长轮询，存在更新延迟或一致性缺失问题。本需求旨在建立一套高性能的客户端缓存同步与一致性检查机制，通过版本控制、差分更新和定期校验，确保客户端数据与服务端基准状态保持强一致或最终一致，同时减少不必要的流量消耗。

## 技术方案

### 1. 版本控制机制
- 在所有后端资源 API 响应中加入 `X-Data-Version` 头，标识数据版本。
- 客户端在本地缓存中记录每个资源 ID 及其关联的 `Version`。

### 2. 差分更新服务
- 在 `gateway` 层实现差分补丁生成服务。客户端请求时带上当前版本号，服务端若有变更，计算并返回增量补丁（JSON Patch 或类似）。
- 如果版本差距过大或无法补丁，则下发完整数据。

### 3. 一致性检查后台检查器
- 实现一个低优先级的后台服务，在游戏静默状态下执行“版本校验”。
- 通过轻量级的 HEAD 请求校验 `X-Data-Version`，仅在不一致时进行同步。

## 验收标准

- [ ] 实现 `X-Data-Version` 全局协议并集成至现有 API。
- [ ] 客户端支持增量补丁解析与应用，流量下降 30% 以上。
- [ ] 后端一致性检查器能自动检测并修复本地缓存数据异常。
- [ ] 所有变更经过单元测试和集成测试覆盖。
