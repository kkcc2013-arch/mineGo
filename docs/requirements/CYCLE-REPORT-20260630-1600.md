### REQ-00389：API 请求速率限制绕过检测与防护系统

- **类别**：安全加固
- **优先级**：P1

#### 背景
当前项目已实现自适应 API 限流（REQ-00098）和动态配额分配系统（REQ-00367），但这些系统存在绕过风险：

- 限流配置探测：逐步增加频率探测限流阈值

#### 目标
构建智能的限流绕过检测与防护系统：
- 识别分布式/协同式绕过行为

- 提供绕过行为可视化与告警

## 🔨 二、需求实现

### REQ-00376：跨区域灾备自动化切换系统（P0）

#### 实现内容

**1. 核心模块**
- `DisasterRecoveryEngine.js` - 灾备自动决策与切换引擎
- `FailoverController.js` - 切换控制器
- `GSLBController.js` - DNS流量控制器
- `PostgreSQLReplicationManager.js` - PostgreSQL复制管理
- `RedisGeoReplication.js` - Redis跨区域复制
- `KafkaMirrorMaker.js` - Kafka镜像同步
- `DatabaseSync.js` - 数据库同步验证
- `DrillManager.js` - 演练管理器

**2. 新增文件**
- `backend/gateway/src/routes/admin/disasterRecovery.js` - 灾备管理API
- `database/migrations/20260630160000_disaster_recovery_tables.sql` - 数据库迁移
- `infrastructure/k8s/monitoring/grafana-dashboard-disaster-recovery.json` - Grafana仪表板

#### 功能亮点

**多维度健康监控**
- 服务可用性（权重30%，阈值95%）
- 数据库健康（权重25%，阈值90%）
- 缓存健康（权重15%，阈值85%）
- 网络延迟（权重15%，阈值100ms）
- 错误率（权重15%，阈值5%）

**自动切换决策**
- 立即切换：健康分数 < 50 持续30秒
- 降级切换：健康分数 < 70 持续60秒且备区域分数 > 90
- 预测切换：预测故障概率 > 80%

**7步切换流程**
1. 停止流量入口
2. PostgreSQL主从切换
3. Redis故障切换
4. 更新服务配置
5. 验证备区域服务
6. 开放备区域流量
7. 验证流量正常

**RTO/RPO目标**
- RTO：5分钟（300秒）
- RPO：1分钟（60秒）
- 实时监控复制延迟

**演练机制**
- 桌面推演（tabletop）
- 模拟演练（simulation）
- 实际切换（full_failover）

#### API接口（12个）

| 接口 | 功能 |
|------|------|
| GET /admin/disaster-recovery/status | 获取灾备整体状态 |
| GET /admin/disaster-recovery/regions | 获取所有区域健康状态 |
| GET /admin/disaster-recovery/regions/:code/health | 获取区域健康详情 |
| POST /admin/disaster-recovery/regions/:code/health | 手动更新区域健康 |
| POST /admin/disaster-recovery/switch | 手动触发灾备切换 |
| POST /admin/disaster-recovery/rollback | 回滚到原主区域 |
| POST /admin/disaster-recovery/drill | 启动灾备演练 |
| GET /admin/disaster-recovery/drills | 获取演练历史 |
| GET /admin/disaster-recovery/history | 获取切换历史记录 |
| GET /admin/disaster-recovery/config | 获取区域配置 |
| PUT /admin/disaster-recovery/config/:code | 更新区域配置 |
| GET /admin/disaster-recovery/metrics | Prometheus指标导出 |
