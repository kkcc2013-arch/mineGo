## 第一步：生成新需求

### REQ-00302: API 分页与列表响应标准化系统

- **类别**: API 设计规范（第 7 类）
- **优先级**: P1
- **状态**: new
- **涉及服务**: gateway、所有微服务、backend/shared/middleware、docs/api-spec

#### 核心内容

1. **统一分页参数规范**
   - 偏移分页：limit/offset 或 page/pageSize
   - 游标分页：cursor/after（适用于大数据集）

2. **统一列表响应格式**
   - data: 数据数组
   - pagination: 分页元数据
   - links: HATEOAS 链接

3. **性能优化**
   - 延迟关联查询（偏移 > 1000 时自动启用）
   - count 查询优化（支持估算模式）
   - 游标分页实现

4. **中间件支持**
   - offsetPaginationMiddleware
   - cursorPaginationMiddleware
   - 自动注入 req.pagination

#### 工作量估算
L (Large) - 预计 16-20 小时

## 第二步：实现未完成需求

### REQ-00294: 动态本地化与玩家语言自适应系统

- **类别**: 国际化/本地化
- **优先级**: P1
- **状态**: done ✅
- **涉及服务**: gateway, user-service, pokemon-service, location-service, reward-service, game-client, backend/shared/i18n

#### 实现清单

##### 1. 语言切换中间件 (localeMiddleware.js)
- 支持 5 种语言：zh-CN, zh-TW, en-US, en-GB, ja-JP
- 多级语言检测（查询参数 > Header > 用户设置 > IP 地区）
- 用户语言偏好持久化
- 实时语言切换，无需重启

##### 2. 翻译缓存系统 (translationCache.js)
- 三级缓存架构：本地缓存 → Redis → PostgreSQL
- 版本管理，支持热更新
- 批量翻译查询优化
- 自动缓存清理机制

##### 3. 区域化适配引擎 (regionalAdapter.js)
- 日期时间格式化（支持时区）
- 数字格式化
- 货币格式化与汇率转换
- 相对时间显示
- 列表格式化

##### 4. 机器翻译服务 (machineTranslation.js)
- Google Translate API 集成
- DeepL API 集成
- 智能提供商选择
- 批量翻译支持
- 翻译缓存

##### 5. 翻译覆盖率监控 (coverageMonitor.js)
- 翻译覆盖率统计
- 缺失翻译检测
- 玩家反馈收集
- Prometheus 指标导出
- 自动质量验证

##### 6. 数据库迁移 (055_i18n_dynamic_localization.sql)
- translation_keys 表
- translations 表
- translation_feedback 表
- user_locale_preferences 表
- 初始化中英日三语翻译数据

##### 7. API 路由 (i18n.js)
- GET /api/v1/i18n/translations/:locale
- POST /api/v1/i18n/translations/reload
- GET /api/v1/i18n/coverage
- POST /api/v1/i18n/feedback
- POST /api/v1/i18n/translate
- POST /api/v1/i18n/format
