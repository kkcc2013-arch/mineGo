- 类别：性能优化
- 优先级：P1
- 状态：done
- 涉及模块：infrastructure/service-registry、gateway、backend/shared/healthCheck

**实现内容**：

#### 核心模块（11,416 行代码）
1. **HealthChecker.js** (7,906 字节)
   - HTTP/HTTPS 健康探针
   - 可配置检查频率（默认 5 秒）、超时（默认 2 秒）、失败阈值（默认 3 次）
   - 连续成功/失败计数与状态转换
   - 事件驱动通知（registered、deregistered、statusChange）

2. **ServiceRegistry.js** (7,984 字节)
   - 服务实例注册/注销
   - 心跳上报机制（默认 10 秒）
   - Redis 持久化支持（TTL 30 秒）
   - 4 种负载均衡策略：round-robin、random、least-connections、weighted

3. **LoadBalancer.js** (6,211 字节)
   - 与健康检查器集成
   - 自动权重调整（基于响应时间）
   - 连接追踪与最少连接策略
   - 事件驱动实例状态更新

#### API 端点
- GET /api/service-discovery/services - 获取所有服务状态
- GET /api/service-discovery/services/:name - 获取单个服务详情
- POST /api/service-discovery/register - 注册新实例
- DELETE /api/service-discovery/services/:instanceId - 注销实例
- PUT /api/service-discovery/services/:instanceId/weight - 更新权重
- GET /api/service-discovery/health - 健康检查状态
- POST /api/service-discovery/health/:instanceId/check - 手动检查
- GET /api/service-discovery/load-balancer/:serviceName - 负载均衡统计

#### 数据库设计
- `service_instances` 表 - 服务实例注册（4 个索引）
- `health_check_history` 表 - 健康检查历史（3 个索引）
- `load_balancer_stats` 表 - 负载均衡统计（1 个索引）
