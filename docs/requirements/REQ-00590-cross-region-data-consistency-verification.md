## 需求描述

为了保证游戏在多地理区域部署时，玩家数据（尤其是跨服交易和排行榜信息）在不同数据中心的一致性，需要构建一套自动化的跨区域数据一致性校验系统。系统需定期对比不同区域的数据副本，发现差异并自动标记，严重时触发告警。

## 技术方案

### 1. 数据校验对比引擎
- 实现基于 Merkle Tree 的数据块对比算法，以最小化跨区传输的数据量。
- 采用非阻塞式定时扫描（Cron 任务），避开高峰期。

### 2. 差异记录与自动修复
- 将差异数据记录至独立的 audit-log 库。
- 对于轻微非关键数据，支持基于 last-write-wins 策略的自动合并；对于关键数据（如货币），仅触发告警待人工介入。

## 验收标准

- [ ] 实现跨区数据校验任务定时调度。
- [ ] 校验任务对线上性能影响小于 2%。
- [ ] 发现不一致数据时能够生成准确的差异报告并触发告警。
- [ ] 提供基础的差异数据可视化仪表盘。

## 影响范围

- database-replica: 新增校验插件。
- cross-region-sync-manager: 新增监控与校验逻辑。
