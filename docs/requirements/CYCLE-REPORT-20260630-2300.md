- 缺少历史版本比对和 Breaking Change 检测

**预期收益**：
- 减少 API Breaking Change 导致的前端故障 **90%**
- 缩短回归测试时间 **50%**
- 支持 50+ 核心 API 的快照覆盖
- CI/CD 集成门禁，防止意外变更

**技术方案**：
- 快照捕获框架（支持 GET/POST/PUT/DELETE）

### REQ-00386：API 响应格式标准化系统

**解决痛点**：
- 响应结构不统一（catch-service vs 其他服务）
- 错误码混乱（HTTP 状态码使用不一致）
- 分页格式多样（多种不同的分页结构）
- 国际化支持缺失（错误消息硬编码）

**实现成果**：
- ✅ **45+ 统一错误码**：涵盖用户、精灵、捕捉、道馆、社交、支付、系统
- ✅ **统一响应格式**：`{ success, data, meta }` 或 `{ success, data, pagination, meta }`
- ✅ **统一错误格式**：`{ success: false, error: { code, message, details, i18nKey, docUrl }, meta }`
- ✅ **完整测试**：17 个单元测试全部通过
- ✅ **详细文档**：API 规范、迁移指南、最佳实践

**技术实现**：
- ErrorCodes.js：错误码分类 `{模块}_{动作}_{原因}`
- ApiResponse.js：工具类方法 success、created、paginated、list、noContent
- errorHandler.js：AppError 类、统一错误处理中间件、asyncHandler
