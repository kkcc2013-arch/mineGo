## 需求描述

针对 mineGo 复杂业务场景（战斗、社交、精灵状态同步），数据库负载持续上升。当前慢查询仅依赖报警，缺乏自动化调优机制。本需求旨在建立一套慢查询监控、分析、建议到自动执行的闭环系统，提升数据库响应速度。

## 技术方案

### 1. 慢查询捕获与标准化
- 集成 `slow_query_log` 或 `pg_stat_statements`，捕获执行时间超过 200ms 的查询。
- 对查询 SQL 进行参数规范化（Normalization），过滤占位符，统一归并。

### 2. 自动化分析与诊断
- 自动分析 `EXPLAIN ANALYZE` 执行计划。
- 识别常见性能瓶颈：全表扫描、索引失效、数据倾斜、锁等待。
- 引入规则引擎对异常查询给出建议（如添加索引、重写 JOIN、分区优化）。

### 3. 自动调优建议与发布
- 集成 GitHub Issues/PR 工作流，自动提交索引创建或优化 PR。
- 对低风险变更（如添加单字段索引）提供"一键预发布"执行接口。

## 验收标准

- [ ] 慢查询捕获率 > 99%，归并准确率 > 95%。
- [ ] 慢查询分析引擎能够识别至少 5 种典型性能瓶颈（全表扫描、缺乏索引等）。
- [ ] 能够生成包含改进建议的报告并推送到 Admin 仪表板。
- [ ] 低风险优化方案支持一键执行，并有回滚方案。
