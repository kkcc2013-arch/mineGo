## 需求描述

为了解决分布式链路中由于微服务抖动导致的性能突降，需要建立一套针对API响应时间(p99, p95)抖动的实时监控与智能告警系统。系统应能够自动基准测试历史抖动模式，识别非预期的异常波动。

## 技术方案

### 1. 监控数据采集
- 在 API 网关和内部微服务通信链路中使用 OpenTelemetry 收集请求耗时。
- 汇总数据到 `metrics-collector` 并计算滑动窗口下的响应时间统计值。

### 2. 智能告警引擎
- 引入基准基线算法（如 Holt-Winters 或简单的 z-score）来识别异常抖动。
- 支持基于业务影响范围的阈值自动调整。

## 验收标准

- [ ] 能够实时可视化展示各服务的 p99/p95 延迟曲线。
- [ ] 针对异常波动（超过基线 3σ）能在 30 秒内触发告警。
- [ ] 告警信息包含相关的链路追踪 ID (Trace ID)。

## 影响范围
