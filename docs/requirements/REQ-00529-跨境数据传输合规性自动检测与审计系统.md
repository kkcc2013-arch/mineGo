# REQ-00529：跨境数据传输合规性自动检测与审计系统

- **编号**：REQ-00529
- **类别**：合规/隐私
- **优先级**：P1
- **状态**：new
- **涉及服务/模块**：gateway、user-service、backend/shared/crossBorderCompliance、backend/jobs/auditJob、admin-dashboard
- **创建时间**：2026-07-11 04:40
- **依赖需求**：REQ-00089（数据跨境传输合规）

## 1. 背景与问题

mineGo 作为全球化 AR 手游，用户数据可能在不同国家/地区之间流动。当前存在以下问题：

1. **合规检测缺失**：缺乏自动化的跨境数据传输合规检测机制，无法识别哪些数据正在跨境传输
2. **法规适应性差**：不同国家/地区（如欧盟 GDPR、美国 CCPA、中国网络安全法）对数据出境有不同要求，当前系统无法灵活适配
3. **审计追溯困难**：跨境数据传输缺乏完整的审计日志，无法满足监管机构的合规审查要求
4. **风险预警不足**：对高风险跨境传输（敏感个人数据、未成年人数据）缺乏实时告警

根据 REQ-00089 的基础合规框架，需要构建更完善的自动化检测与审计系统。

## 2. 目标

1. 建立自动化的跨境数据传输检测机制，识别所有跨境数据流
2. 实现多法域合规规则引擎，支持 GDPR、CCPA、中国网络安全法等主流法规
3. 提供完整的跨境传输审计日志和合规报告生成功能
4. 对高风险传输实现实时告警和阻断能力

## 3. 范围

- **包含**：
  - 数据出境流量分析引擎
  - 多法域合规规则引擎（GDPR、CCPA、中国网安法、PDPA 等）
  - 跨境传输审计日志系统
  - 合规风险评估与预警
  - 合规报告自动生成
  - Admin Dashboard 合规监控页面

- **不包含**：
  - 数据本地化存储实施（已有需求覆盖）
  - 具体法规的法律解读（由法务团队维护规则库）

## 4. 详细需求

### 4.1 数据出境流量分析引擎

**4.1.1 流量识别**
- 拦截所有服务间通信，识别源和目标的国家/地区
- 支持识别数据类型：用户身份信息、位置数据、支付信息、设备信息、游戏数据
- 通过 IP 地理位置库（MaxMind GeoIP2）确定数据流向

### 4.2 多法域合规规则引擎

**4.2.1 法规规则库**
```javascript
const complianceRules = {
  GDPR: {
    region: 'EU',
    rules: [
      {
        id: 'GDPR_ART44',
        description: '传输至第三国需确保充分保护',
        check: async (transfer) => {
          // 检查目标国是否在欧盟充分性认定名单
          // 或有标准合同条款（SCCs）
          // 或有约束性企业规则（BCRs）
          return targetCountryInAdequacyList(transfer.targetCountry) ||
                 hasStandardContractualClauses(transfer) ||
                 hasBindingCorporateRules(transfer);
        },
        severity: 'CRITICAL'
      },
      {
        id: 'GDPR_ART49',
        description: '特殊情况下可豁免充分保护要求',
        check: async (transfer) => {
          // 用户明确同意、履行合同必需等
          return hasExplicitConsent(transfer) ||
                 isContractuallyNecessary(transfer);
        },
        severity: 'HIGH'
      }
    ]
  },
  
  CHINA_CSL: {
    region: 'CN',
    rules: [
      {
        id: 'CSL_ART37',
        description: '关键信息基础设施运营者数据出境需安全评估',
        check: async (transfer) => {
          if (isCIIOperator(transfer.sourceOrg)) {
            return hasSecurityAssessment(transfer);
          }
          return true;
        },
        severity: 'CRITICAL'
      },
      {
        id: 'CSL_DATA37',
        description: '个人信息出境需通过安全评估或签订标准合同',
        check: async (transfer) => {
          const personalDataCount = transfer.personalDataCount;
          if (personalDataCount > 100000) {
            return hasCACSecurityAssessment(transfer);
          } else {
            return hasStandardContract(transfer);
          }
        },
        severity: 'HIGH'
      }
    ]
  },
  
  CCPA: {
    region: 'US_CA',
    rules: [
      {
        id: 'CCPA_SALE_OPTOUT',
        description: '加州用户有权拒绝数据出售',
        check: async (transfer) => {
          if (transfer.isSale === true) {
            return !userHasOptedOut(transfer.userId);
          }
          return true;
        },
        severity: 'HIGH'
      }
    ]
  },
  
  PDPA_SG: {
    region: 'SG',
    rules: [
      {
        id: 'PDPA_TRANSFER',
        description: '传输至新加坡境外需确保同等保护',
        check: async (transfer) => {
          return hasBindingContract(transfer) ||
                 isInWhiteList(transfer.targetCountry);
        },
        severity: 'HIGH'
      }
    ]
  }
};
```

**4.2.2 规则评估流程**
1. 根据数据源地区确定适用法规
2. 逐一检查相关规则
3. 生成合规评估结果（COMPLIANT / NON_COMPLIANT / REVIEW_REQUIRED）
4. 对不合规传输生成整改建议

### 4.4 风险评估与预警

**4.4.1 风险等级计算**
```javascript
const riskFactors = {
  dataSensitivity: {
    CRITICAL: 40,  // 支付信息、未成年人数据
    HIGH: 30,      // 身份信息、位置数据
    MEDIUM: 20,    // 设备信息
    LOW: 10        // 游戏数据
  },
  
  transferVolume: {
    MASSIVE: 30,   // >100万条
    LARGE: 20,     // 10万-100万条
    MEDIUM: 15,    // 1万-10万条
    SMALL: 5       // <1万条
  },
  
  targetJurisdiction: {
    HIGH_RISK: 25, // 无数据保护法的国家
    MEDIUM_RISK: 15, // 有数据保护法但保护水平较低
    LOW_RISK: 5    // 保护水平与本国相当或更高
  },
  
  userConsent: {
    NONE: 20,      // 无用户同意
    IMPLIED: 10,   // 默示同意
    EXPLICIT: 0    // 明确同意
  }
};

function calculateRiskLevel(transfer) {
  const score = 
    riskFactors.dataSensitivity[transfer.sensitivityLevel] +
    riskFactors.transferVolume[transfer.volumeLevel] +
    riskFactors.targetJurisdiction[transfer.jurisdictionRisk] +
    riskFactors.userConsent[transfer.consentLevel];
  
  if (score >= 80) return 'CRITICAL';
  if (score >= 60) return 'HIGH';
  if (score >= 40) return 'MEDIUM';
  return 'LOW';
}
```

**4.4.2 预警机制**
- CRITICAL 风险：立即阻断传输，发送紧急告警
- HIGH 风险：允许传输但记录告警，24小时内人工复核
- MEDIUM 风险：记录日志，周报汇总
- LOW 风险：仅记录

### 4.5 合规报告生成

**4.5.1 报告类型**
1. **实时合规看板**：展示当前跨境传输状态、合规率、风险分布
2. **月度合规报告**：汇总月度传输情况、合规趋势、主要风险点
3. **年度审计报告**：符合监管要求的完整审计文档
4. **事件响应报告**：针对特定合规事件的专项报告

**4.5.2 报告内容**
- 传输统计：数量、数据量、地区分布
- 合规情况：合规率、违规详情、整改情况
- 风险分析：风险等级分布、主要风险因素
- 法规遵从：各项法规的遵从情况
- 改进建议：基于分析结果的优化建议

## 7. 优先级理由

**P1 理由**：
1. **合规强制**：跨境数据传输是全球化应用的强制合规要求，违规可能导致巨额罚款（如 GDPR 最高 2000 万欧元）
2. **监管审查**：近期多国加强对数据出境的监管，需要有完善的审计能力应对检查
3. **风险控制**：对高风险传输的实时阻断能力可避免重大合规事件
4. **用户信任**：完善的数据保护措施有助于提升用户信任度

这是项目"可用"的关键合规能力，属于基础设施类需求。
