## 需求描述

当前数据库连接池主要为静态配置，难以应对流量突发或低谷期的情况。本项目需要实现一个动态连接池调整机制，能够根据请求延迟、数据库负载指标（CPU/IO）以及连接池利用率自动增减连接数量，以提升资源利用率并降低延迟。

## 技术方案

### 1. 监控与指标采集
- 引入监控 Sidecar 实时采集连接池状态（活跃连接数、闲置连接数、等待队列长度）。
- 整合 Prometheus 监控 DB 实例负载。

### 2. 自适应调整算法
- 基于 PI 控制算法或简单的阈值触发机制，动态计算最优池大小。
- 引入最大最小连接数约束，防止过激的扩缩容导致数据库抖动。

### 3. 热配置变更
- 使用配置中心（如 Apollo 或 ConfigMap 热更新）动态调整动态伸缩策略的敏感参数。

## 验收标准

- [ ] 能够在高负载场景下自动扩容，等待队列时间保持在 50ms 以下。
- [ ] 在低流量期能够平滑缩容以释放 DB 资源。
- [ ] 调整过程中不影响现有长连接业务。
- [ ] 提供实时连接池状态看板。
