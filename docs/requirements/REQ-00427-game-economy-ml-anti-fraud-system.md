## 需求描述

为了更有效地识别游戏经济系统中的新型作弊行为（如工作室自动化挂机、非法刷币、多账号倒卖资源），现有的基于规则的反作弊系统（REQ-00416）需要升级为基于机器学习的动态预测系统。该需求旨在建立一套自动化的训练、评估、发布流水线，基于实时业务行为日志，定期生成并更新欺诈检测模型。

## 技术方案

### 1. 模型训练流水线 (Data Pipeline)
- **数据源**：从 Data Lake (HDFS/S3) 抽取最近 30 天的高价值交易数据、账号登录行为、社交互动日志。
- **特征工程**：自动计算玩家活跃度、资源获取速度、交易频率、关联账号网络图特征。
- **训练引擎**：使用 Spark MLlib 或 Kubeflow Pipeline 执行自动模型训练。

### 2. 模型评估与发布 (Model Serving)
- **离线评估**：自动计算模型的召回率、准确率，并与基准模型对比。
- **金丝雀发布**：模型通过 gRPC 请求进入 `shared/risk-engine` 服务，支持同时运行多个版本，并根据路由规则进行流量切换。
- **监控**：集成 Prometheus 监控实时检测模型的 FP（误报）与 FN（漏报）率。

## 验收标准

- [ ] 实现反欺诈特征自动化提取与入库流程
- [ ] 搭建基于 Kubeflow 的模型自动训练调度周期（每日/每周）
- [ ] 支持反欺诈模型版本化发布与回滚机制
- [ ] 实现模型性能指标仪表板（召回率、误报率、预测耗时）
- [ ] 系统能自动检测到特定作弊行为并触发警报

## 影响范围

- `shared/risk-engine`：模型推理逻辑
- `admin-dashboard`：模型状态管理与控制台
- `data-service`：数据聚合与特征存储
