# REQ-00474: 游戏色彩感知障碍辅助与自定义调色板系统

- **编号**：REQ-00474
- **类别**：无障碍(a11y)
- **优先级**：P2
- **状态**：new
- **涉及服务/模块**：game-client、shared/color-system、admin-dashboard
- **创建时间**：2026-07-07 06:00
- **依赖需求**：REQ-00059（新手引导系统）, REQ-00435（触觉反馈增强）

## 1. 背景与问题

mineGo 作为一款基于真实 GPS 的 AR 精灵捕捉手游，游戏界面大量依赖色彩来传递关键信息：
- 精灵属性：火系（红）、水系（蓝）、草系（绿）、电系（黄）等 18 种属性各有专属颜色
- 稀有度标识：普通（灰）、稀有（蓝）、史诗（紫）、传说（金）
- 战斗状态：HP 血条（绿→黄→红）、增益效果（蓝）、减益效果（红）
- 地图标记：道馆（蓝色）、补给站（紫色）、精灵刷新点（黄色）

**问题分析：**
1. **色觉障碍群体被忽视**：全球约 8% 的男性（约 3 亿人）存在不同程度的色觉障碍（红绿色盲最常见）
2. **信息传递单一路径**：关键状态仅通过颜色传递，缺少形状、图案等替代方案
3. **无法自定义配色**：玩家无法根据自身视觉特点调整游戏界面颜色
4. **不符合 WCAG 2.1 AA 标准**：未达到无障碍设计最低要求，影响产品国际化和合规性
5. **竞争产品已实现**：Pokémon GO、Ingress 等竞品已提供色盲模式

## 2. 目标

实现完整的色彩感知障碍辅助系统，确保所有视觉信息都有非色彩替代方案，并提供自定义调色板功能。

**量化目标：**
- 覆盖 100% 的关键颜色编码信息（精灵属性、稀有度、状态等）
- 提供至少 5 种预设色盲模式（红绿色盲、蓝黄色盲、全色盲等）
- 支持玩家自定义调色板（每类元素可单独调整颜色）
- WCAG 2.1 AA 合规率 100%
- 无障碍模式启用率目标：10%+ 的活跃玩家

## 3. 范围

**包含：**
- 色盲模式预设（红绿色盲、蓝黄色盲、全色盲、单色觉）
- 每个元素的替代性视觉标识（形状、图案、文字标签）
- 自定义调色板编辑器（精灵属性、稀有度、状态条、地图标记）
- 色彩对比度检测器（自动检测不合规颜色组合）
- 高对比度模式（增强边缘和文字对比度）
- 设置界面与实时预览

**不包含：**
- 完全重构 UI（仅在现有界面添加辅助层）
- 语音辅助功能（属于 REQ-00435）
- 其他无障碍功能（如键盘导航、屏幕阅读器）

## 4. 详细需求

### 4.1 色盲模式预设系统

实现 5 种预设模式：
1. **标准模式**：正常色彩显示
2. **红色盲模式（Protanopia）**：调整红色相关颜色，增强对比度
3. **绿色盲模式（Deuteranopia）**：调整绿色相关颜色，使用青蓝替代
4. **蓝黄色盲模式（Tritanopia）**：调整蓝黄色系，增强区分度
5. **全色盲模式（Achromatopsia）**：灰度显示 + 图案标识

### 4.2 形状与图案标识系统

为 18 种精灵属性分配独特形状：
- 火系：三角形（▲）
- 水系：水滴形（💧）
- 草系：叶形（🍃）
- 电系：闪电形（⚡）
- 冰系：菱形（◆）
- 格斗系：拳头形（👊）
- 毒系：骷髅形（☠）
- 地面系：方形（■）
- 飞行系：羽毛形（🪶）
- 超能系：眼形（👁）
- 虫系：六边形（⬡）
- 岩石系：五边形（⬠）
- 幽灵系：新月形（🌙）
- 龙系：星形（★）
- 恶系：月亮形（🌑）
- 钢系：齿轮形（⚙）
- 妖精系：心形（♥）
- 普通系：圆形（●）

### 4.3 自定义调色板编辑器

- 颜色选择器：支持所有游戏元素的颜色自定义
- 实时预览：修改颜色时立即看到效果
- 对比度检查：自动检测颜色对比度是否符合 WCAG 标准
- 重置功能：一键恢复默认颜色

### 4.4 状态条视觉增强

HP 血条分段标识：
- 健康（>50%）：纯色 + 实心
- 中度（25-50%）：条纹图案
- 危险（<25%）：点状图案 + 闪烁动画
- 濒危（<10%）：危险条纹 + 高频闪烁

## 7. 优先级理由

**P2 理由：**
1. 影响约 8% 男性用户（潜在 3 亿人），用户基数大
2. 符合国际无障碍标准，有利于产品全球化
3. 提升品牌形象，体现社会责任
4. 竞品已实现，属于市场基本要求
5. 不影响核心功能，可延后实现

## 8. 技术实现要点

- 使用 CSS 变量实现颜色系统，便于动态修改
- SVG 图形生成形状标识，保证缩放清晰
- localStorage 持久化用户偏好设置
- Brettel 算法实现色盲模拟
- WCAG 对比度计算算法验证颜色合规性

## 9. 用户测试计划

- 招募 10 名色觉障碍用户进行可用性测试
- 测试场景：精灵识别、战斗状态判断、地图导航
- 收集反馈并优化颜色方案
- A/B 测试启用率与用户满意度
