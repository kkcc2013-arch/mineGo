## 任务1：生成新需求 REQ-00559

### 需求信息

- **编号**: REQ-00559
- **标题**: 数据库连接池智能预热与自适应管理系统
- **类别**: 性能优化
- **优先级**: P1
- **涉及服务**: backend/shared/db, 所有微服务

### 需求背景

mineGo 项目采用 PostgreSQL 15 作为主数据库，所有微服务通过 `backend/shared/db` 模块共享数据库连接池。当前实现存在以下性能问题：

1. **冷启动延迟**: 服务重启后首次查询响应时间高达 500-2000ms
2. **连接池管理粗放**: 使用固定连接池大小，未根据负载动态调整
3. **健康检查缺失**: 未检测连接池实际可用性，异常无自动恢复
4. **与业务场景脱节**: 未结合用户活跃时段预预热

### 需求目标

构建智能的数据库连接池管理系统：

1. 智能预热：服务启动时自动预热，首次查询延迟降低80%
2. 自适应调整：根据负载动态调整，资源利用率提升40%
3. 健康监控：实时监控，异常自动恢复，故障率降低90%
4. 时段调度：结合业务高峰时段预扩容

## 任务2：实现 REQ-00559

### 实现架构

采用三层架构设计：

```
┌─────────────────────────────────────────────────────────────┐
│                IntelligentPoolManager                        │
│  (统一管理接口 - 初始化、状态查询、管理API)                   │
└─────────────────────────────────────────────────────────────┘
                           │
          ┌────────────────┼────────────────┐
          ▼                ▼                ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│  PoolPreheater  │ │PoolHealthChecker│ │AdaptivePool     │
│  (连接池预热)    │ │  (健康检查)      │ │  Manager        │
│                 │ │                 │ │  (自适应调整)    │
└─────────────────┘ └─────────────────┘ └─────────────────┘
```

### 核心模块

#### 1. PoolPreheater.js (11.4 KB)

**功能**:
- 启动预热：服务启动时创建最小连接数
- 时段调度：根据高峰时段配置自动预扩容
- 手动预热：支持管理员手动触发
- 活动预热：支持运营活动专项预热

**关键方法**:
- `preheatOnStartup()`: 启动预热
- `preheatForPeak(peak)`: 高峰时段预热
- `manualPreheat(connections)`: 手动预热
- `preheatForEvent(eventConfig)`: 活动预热

#### 2. PoolHealthChecker.js (8.6 KB)

**功能**:
- 定期健康检查：每30秒检测连接池可用性
- 连接泄漏检测：识别持有时间过长的连接
- 自动恢复：异常时自动重试恢复（最多3次）
- Prometheus指标：实时输出健康状态

**关键方法**:
- `check()`: 执行健康检查
- `_detectLeaks()`: 检测连接泄漏
- `_attemptRecovery(error)`: 尝试恢复

#### 3. IntelligentPoolManager.js (12.2 KB)

**功能**:
- 统一管理：整合预热、健康检查、自适应调整
- 自动扩缩容：基于负载动态调整连接池大小（5-50）
- 时段感知：根据时间段应用不同策略
- 管理API：提供手动操作接口

**关键方法**:
- `initialize()`: 初始化所有子系统
- `getStatus()`: 获取综合状态
- `manualPreheat()`: 手动预热
- `resizePool(newSize)`: 调整池大小

### Prometheus 监控指标

新增 9 个核心指标：

1. `db_pool_healthy` - 健康状态（0/1）
2. `db_pool_total_connections` - 总连接数
3. `db_pool_idle_connections` - 空闲连接数
4. `db_pool_waiting_connections` - 等待连接数
5. `db_pool_utilization` - 使用率
6. `db_pool_current_size` - 当前目标大小
7. `db_pool_leak_detected` - 泄漏连接数
8. `db_pool_preheated` - 预热状态（0/1）
9. (复用) `db_pool_scale_up_total` - 扩容总次数
10. (复用) `db_pool_scale_down_total` - 缩容总次数
