## 需求描述

为了解决数据库读写分离后的数据不一致问题，需要实现一套读写分离监控与延迟告警系统。该系统需监测主从数据库之间的复制延迟（Replication Lag），并在延迟超过阈值时触发告警，同时提供延迟趋势分析与历史监控数据，以便优化数据库配置。

## 技术方案

### 1. 监控采集模块
- 开发监控 Sidecar 程序，周期性在从节点执行 `SHOW SLAVE STATUS` (或对应数据库监控命令)。
- 计算延迟时间（单位：ms），并将指标推送到 Prometheus。

### 2. 告警管理模块
- 定义不同的告警等级：
    - P2: 延迟 > 500ms（警告）
    - P1: 延迟 > 2s（严重）
    - P0: 延迟 > 5s（关键，自动将相关请求熔断至主库）

### 3. 数据可视化
- 在 Grafana 仪表板中添加“读写分离复制状态”监控项。
- 提供复制延迟趋势图、主从状态概览。

## 验收标准

- [ ] 成功在 Prometheus 中采集到各从库的复制延迟指标。
- [ ] 配置完善的告警规则，能够区分不同等级的延迟。
- [ ] 延迟超过阈值时能及时发出告警。
- [ ] Grafana 仪表板包含延迟趋势看板。
