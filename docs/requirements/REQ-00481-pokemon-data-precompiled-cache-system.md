## 需求描述

为了解决精灵数据（包括属性、进化链、技能集、获取路径等）在高并发查询下的频繁计算与反序列化开销，设计一套预编译缓存系统。该系统在系统启动或数据更新时，将复杂的原始精灵配置数据直接编译为高效的二进制或缓存友好的内存结构，并同步至 Redis 分布式缓存中。

## 技术方案

### 1. 预编译层
- 使用 MessagePack 或 Protobuf 格式对精灵数据进行高效序列化。
- 实现 `PokemonDataCompiler` 工具类，支持在编译阶段完成基础统计计算（如满级属性预估、进化节点查找）。

### 2. 缓存分层策略
- **L1 (Local Cache):** 使用 `LRU` 缓存存储最常用的精灵基础数据。
- **L2 (Redis):** 分布式缓存存储序列化后的预编译二进制块，通过一致性哈希分布，支持按需加载。

### 3. 数据更新同步
- 监听 `data-config-change` 事件，自动触发缓存失效与更新。
- 使用版本号机制，确保客户端缓存与后端配置同步。

## 验收标准

- [ ] 精灵基础数据查询延迟降低至 5ms 以内。
- [ ] 支持在不重启服务的情况下触发数据预编译与更新。
- [ ] 缓存失效策略能够确保数据一致性，更新后 1s 内全网生效。
- [ ] 新增单元测试，验证序列化与反序列化的性能提升。

## 影响范围

- `pokemon-service` 核心读接口
- `data-config` 模块
- Redis 缓存结构
