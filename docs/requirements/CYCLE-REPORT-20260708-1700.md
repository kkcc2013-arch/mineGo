## 一、新增需求

### REQ-00509: CI/CD 缓存智能管理与优化系统

| 字段 | 值 |
|------|-----|
| 编号 | REQ-00509 |
| 类别 | 运维/CICD |
| 优先级 | P2 |
| 状态 | new |
| 涉及模块 | .github/workflows、backend/shared/cacheManager、infrastructure/cache |

**需求背景**：
- 当前 CI/CD 缓存策略分散在各工作流中
- 缓存命中率偏低，缺乏统一管理
- 缓存过期无自动清理机制
- 跨工作流缓存共享不足

**核心功能**：
- 缓存键值统一设计（基于 package-lock.json、Dockerfile 指纹）
- 缓存命中率统计与分析服务
- 缓存清理策略引擎（LRU + 过期时间 + 手动标记）
- Admin Dashboard 缓存管理界面

## 二、实现需求

### REQ-00502: 性能分析与深度优化框架设计 ✅ done

**实现文件**：

| 文件 | 说明 |
|------|------|
| `backend/shared/PerformanceSampler.js` | 采样核心模块（12889 字节） |
| `gateway/src/middleware/performanceSampling.js` | Gateway 中间件（4310 字节） |
| `backend/shared/PerformanceAnalyzer.js` | 热点分析引擎（10656 字节） |
| `backend/jobs/performanceReportGenerator.js` | 定时报告任务（5453 字节） |
| `backend/shared/tests/PerformanceSampler.test.js` | 单元测试（7900 字节） |

**核心功能**：

1. **PerformanceSampler**：
   - 采样率配置（默认 5%，可动态调整）
   - 全链路追踪（auth → validation → db → cache → api → business → response）
   - 数据库操作记录（operation、table、durationMs、query）
   - 缓存操作记录（operation、key、durationMs、hit）
   - API 调用记录（targetService、endpoint、durationMs、statusCode）
   - Prometheus 指标集成（requestDuration、dbDuration、cacheDuration、apiDuration）
   - 热点自动识别（数据库瓶颈、API 瓶颈、缓存瓶颈、业务逻辑瓶颈）

2. **Gateway 中间件**：
   - 请求拦截与响应监听
   - 采样率动态配置接口 `/admin/performance/sampling-rate`
   - 热点分析接口 `/admin/performance/hotspots`
   - Prometheus 指标接口 `/metrics/performance`
   - 统计重置接口 `/admin/performance/reset`

3. **PerformanceAnalyzer**：
   - 分析规则定义（慢查询阈值 100ms、慢 API 500ms、慢缓存 50ms）
   - 瓶颈严重度分级（critical > 2000ms、high > 1000ms、medium > 500ms、low）
   - 批量分析聚合（端点分析、瓶颈分布、问题分布）
   - 优化建议生成（针对性建议）
   - 历史趋势分析（7 天趋势对比）

4. **定时报告任务**：
   - 每日早上 6 点自动生成报告
   - Slack 通知格式化
   - 手动触发接口

**单元测试覆盖**：
- 采样逻辑测试
- 样本追踪测试（startSampling、startPhase、endPhase）
- 数据库操作记录测试
- 缓存操作记录测试（命中/未命中）
- API 调用记录测试
- 热点分析测试（慢端点识别、排序）
- 报告生成测试
- 配置管理测试（采样率更新、统计重置）
