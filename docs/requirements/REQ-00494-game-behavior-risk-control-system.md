## 需求描述

构建一个针对游戏内高频操作、地理位置突变、异常物品交易等行为的实时风控系统。通过大数据分析与机器学习模型，识别潜在的自动化脚本、外挂使用行为，并实现自动化的封禁策略或人工审核上报。

## 技术方案

### 1. 行为数据采集管道
- 利用 Kafka 实时流式传输玩家行为日志 (动作、位置、时间戳)。
- 数据清洗与标准化处理，提取特征序列。

### 2. 实时风险检测引擎
- 规则引擎：处理固定阈值的非法行为（如高频点击、跨区域瞬间移动）。
- 机器学习模型（如异常检测）：基于滑动窗口分析行为特征是否存在作弊指纹。

### 3. 反作弊执行系统
- 支持等级制的应对策略：警告、临时限速、封号、进入观察队列。
- 控逻辑的可配置性，支持热更新规则。

## 验收标准

- [x] 行为数据采集延迟小于 500ms
- [x] 识别准确率达到 95% 以上
- [x] 支持在线封号处理，无需重启服务
- [x] 具备完善的审计日志记录

## 实现详情

### 已实现文件
1. **RiskControlEngine.js** (31,175 字节)
   - Kafka 事件消费引擎
   - 7 个反作弊规则：速度异常、捕捉频率、GPS伪造、道具使用、道馆战斗、多设备登录、异常交易
   - 实时评分系统（0-100分）
   - 4级行动决策（警告、限速、临时封禁、永久封禁）
   - Prometheus 监控指标

2. **数据库架构** (7,803 字节)
   - user_behavior_events（用户行为事件）
   - anti_cheat_audit_logs（审计日志）
   - user_bans（封禁记录）
   - manual_review_queue（人工审核队列）
   - risk_control_rules（规则配置）
   - user_risk_scores（评分历史）
   - user_risk_state（实时状态）

3. **单元测试** (11,841 字节)
   - 覆盖主要检测场景
   - 覆盖边界情况
   - 覆盖数据库操作
