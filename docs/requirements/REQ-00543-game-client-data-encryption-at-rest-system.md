## 需求描述

为了防止玩家在本地设备上的游戏数据（如本地缓存、进度、个人偏好设置）被恶意读取或篡改，需要实现一套游戏客户端本地存储数据加密系统。系统应支持基于设备密钥的 AES-256 加密，确保数据在静态状态下的安全性。

## 技术方案

### 1. 密钥管理
- 使用 Android Keystore / iOS Keychain 安全地存储和检索加密密钥。
- 密钥在首次运行生成，且不可导出。

### 2. 数据加密层
- 在本地存储接口中引入中间件，自动对存入的 JSON 数据进行加密。
- 使用 AES-256 (GCM 模式) 以提供加密和完整性校验。

### 3. 数据恢复与迁移
- 实现加密版本升级策略，支持旧格式数据的平滑迁移（加密）。

## 验收标准

- [ ] 确保所有本地存储文件（除了系统配置文件）均被加密。
- [ ] 验证在设备密钥未改变的情况下，数据读写准确无误。
- [ ] 模拟内存溢出或进程崩溃，验证数据存储原子性。
