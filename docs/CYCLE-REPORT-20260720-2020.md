##### 1. ReadWriteSplitManager（读写分离管理器）

**文件：** `backend/shared/dbReadWriteSplit/ReadWriteSplitManager.js`  
**代码行数：** 13,336 行

**核心功能：**
- 自动读写路由：写操作（INSERT/UPDATE/DELETE）→ 主库，读操作（SELECT）→ 副本
- 负载均衡策略：
  - **Round-Robin（轮询）**：依次选择副本
  - **Weighted（加权）**：按权重随机选择
  - **Least-Connections（最少连接）**：选择连接数最少的副本
- 副本健康检查与自动故障切换
- 延迟阈值管理：
  - warning: 500ms
  - critical: 2000ms（切换到主库）
  - max: 5000ms（副本下线）

**关键代码片段：**

```javascript
// 自动路由
async query(sql, params = [], options = {}) {
  const isWrite = this.isWriteQuery(sql);
  const consistency = this.determineConsistency(sql, options);
  
  if (isWrite || consistency === 'strong') {
    return await this.executeQuery('primary', sql, params, options);
  } else {
    return await this.executeReadQuery(sql, params, options);
  }
}

// 负载均衡
selectReplica() {
  const healthyReplicas = this.replicaPools.filter(r => 
    this.replicaHealth[r.id].healthy && 
    this.lagData[r.id] < this.config.lagThresholds.critical
  );
  
  if (healthyReplicas.length === 0) {
    return null; // fallback to primary
  }
  
  return this.selectRoundRobin(healthyReplicas);
}
```

##### 2. ReplicaLagMonitor（副本延迟监控）

**文件：** `backend/jobs/replicaLagMonitor.js`  
**代码行数：** 9,905 行

**核心功能：**
- 定时心跳检测（默认 5 秒间隔）
- 两种延迟测量方式：
  - **心跳表方式**：在主库写入时间戳，从副本读取并计算延迟
  - **pg_stat_replication**：从 PostgreSQL 系统视图获取延迟
- 延迟告警与分级通知
- Redis 缓存延迟数据

**监控流程：**

```
主库写入心跳 → 副本读取心跳 → 计算延迟 → 更新健康状态 → 触发告警
    ↓              ↓              ↓            ↓            ↓
INSERT          SELECT         lag = t2-t1   health.lag   sendAlert()
```

##### 3. ConsistencyMiddleware（一致性中间件）

**文件：** `backend/gateway/src/middleware/consistencyMiddleware.js`  
**代码行数：** 6,478 行

**核心功能：**
- 强一致性检测：
  - Header: `X-Consistency-Level: strong`
  - 查询参数: `?force_master=true`
  - 路径匹配: `/api/payment`, `/api/trade` 等
- 副本延迟过高时自动 fallback 到主库
- 健康检查端点: `/health/db`
- 延迟数据端点: `/api/admin/replica/lag`
