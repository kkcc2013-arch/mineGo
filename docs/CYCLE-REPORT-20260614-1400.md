## 任务 1：生成新需求

### 新需求信息
- **编号**: REQ-00198
- **标题**: 游戏节奏控制与慢速模式系统
- **类别**: 无障碍(a11y) (类别轮转: 197 % 18 = 17)
- **优先级**: P2
- **涉及服务**: game-client、frontend/game-client/src/game、frontend/game-client/src/accessibility、gateway

### 需求背景
当前 mineGo 游戏客户端缺少游戏节奏控制能力：
- 捕捉机制速度过快，运动障碍用户无法在时间窗口内完成操作
- 战斗系统反应时间要求高，对帕金森病、关节炎患者不友好
- 认知障碍用户面临过重的认知负荷

### 核心功能
1. 捕捉场景时间缩放（投掷窗口延长、摇球动画减速）
2. 战斗系统节奏控制（攻击间隔延长、躲避窗口扩大）
3. UI 动画速度调节
4. 用户偏好持久化

#### 技术亮点
1. **静态代码分析**：
   - HTTP 客户端调用提取（axios, fetch）
   - Kafka 事件发布/订阅提取
   - 环境变量服务名解析
   - Gateway 代理配置解析

2. **算法实现**：
   - **DFS 算法**：检测循环依赖，准确识别回边
   - **Kahn 拓扑排序**：计算最优服务启动顺序

3. **可视化输出**：
   - Mermaid 格式（Markdown 内嵌）
   - GraphViz DOT 格式（高质量 PNG/SVG）

4. **API 端点**（7 个）：
   ```
   GET  /api/admin/dependencies              # 完整依赖图
   GET  /api/admin/dependencies/:service     # 单服务依赖
   GET  /api/admin/dependencies/cycles       # 循环检测
   GET  /api/admin/dependencies/startup-order # 启动顺序
   GET  /api/admin/dependencies/graph        # Mermaid/DOT 图
   GET  /api/admin/dependencies/impact/:service # 故障影响分析
   POST /api/admin/dependencies/refresh      # 强制刷新
   ```

5. **CI/CD 集成**：
   - 自动运行依赖分析
   - 发现循环依赖时构建失败
   - 生成依赖图并上传 artifact
   - PR 评论自动附加依赖图
