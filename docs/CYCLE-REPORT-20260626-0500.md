## 任务详情

### 1. 需求生成

**编号**：REQ-00335  
**标题**：游戏距离单位本地化与智能转换系统  
**类别**：国际化/本地化（类别编号 10，334 % 18 = 10）  
**优先级**：P1

**需求背景**：
- 项目已完成语言（REQ-00011）、时区（REQ-00029）、货币（REQ-00051）本地化
- 距离单位本地化是国际化体系的最后一块拼图
- 美国、利比里亚、缅甸使用英制单位，其他地区使用公制
- 当前距离显示硬编码为公制单位（m、km/h），影响美国用户体验

**解决目标**：
- 支持公制/英制自动切换
- 覆盖距离、速度、温度、重量、面积 5 类物理量
- 根据用户地区或浏览器语言自动推断单位制
- 支持用户手动切换偏好

#### 2.2 实现亮点

1. **智能检测机制**
   ```javascript
   // 检测优先级：localStorage > 浏览器语言 > IP 地理位置 > 默认公制
   async function detectUnitSystem() {
     const saved = localStorage.getItem('pmg_unit_system');
     if (saved) return saved;
     
     const locale = navigator.language;
     if (locale === 'en-US') return 'imperial';
     
     return 'metric';
   }
   ```

2. **统一格式化 API**
   ```javascript
   formatDistance(150)    // 公制: "150 m" | 英制: "492 ft"
   formatDistance(2500)   // 公制: "2.5 km" | 英制: "1.6 mi"
   formatSpeed(10)        // 公制: "36 km/h" | 英制: "22 mph"
   formatTemperature(25)  // 公制: "25°C" | 英制: "77°F"
   ```

3. **数据库迁移自动化**
   ```sql
   -- 自动设置现有用户的单位制偏好
   UPDATE users SET unit_system = 'imperial'
   WHERE country IN ('US', 'LR', 'MM');
   ```

4. **后端 API 设计**
   ```
   PUT  /api/v1/users/me/unit-system  - 更新单位制偏好
   GET  /api/v1/users/me/unit-system  - 获取单位制偏好
   GET  /api/v1/users/me/preferences  - 获取所有偏好（含单位制）
   ```
