# mineGo 错误码参考

> 最后更新：2026-06-05

本文档列出 mineGo API 所有错误码及其含义，帮助开发者快速定位问题。

---

## 错误码范围

| 范围 | 类别 | 说明 |
|------|------|------|
| 1000-1999 | 通用错误 | 参数、认证、权限等通用问题 |
| 2000-2999 | 用户相关 | 注册、登录、资料、货币等 |
| 3000-3999 | 精灵/捕捉 | 精灵、捕捉、补给站等 |
| 4000-4999 | 道馆/社交 | 道馆、好友、礼物、交换、Raid |
| 5000-5999 | 支付 | 订单、商品、支付等 |
| 9000-9999 | 系统错误 | 服务器内部错误 |

---

## 详细错误码

### 通用错误 (1000-1999)

| 错误码 | HTTP 状态 | 说明 | 解决方案 |
|--------|-----------|------|----------|
| 1001 | 400 | 参数错误 | 检查请求参数格式、类型、必填项 |
| 1002 | 401 | 未认证，请先登录 | 使用 `/auth/login` 获取 Token |
| 1003 | 401 | Token 无效或已过期 | 使用 `/auth/refresh` 刷新 Token 或重新登录 |
| 1004 | 403 | 权限不足 | 检查用户权限或资源所有权 |
| 1005 | 404 | 资源不存在 | 确认资源 ID 是否正确 |
| 1006 | 405 | 方法不允许 | 检查 HTTP 方法是否正确 |
| 1007 | 429 | 请求过于频繁，请稍后重试 | 降低请求频率，等待 1 分钟后重试 |
| 1008 | 400 | 验证码错误或已过期 | 重新获取验证码 |
| 1009 | 408 | 请求超时 | 检查网络连接，稍后重试 |
| 1010 | 415 | 内容类型不支持 | 使用 `application/json` |

**示例响应**：

```json
{
  "code": 1003,
  "message": "Token 无效或已过期",
  "data": null,
  "traceId": "abc-123"
}
```

| 3003 | 400 | 精灵球不足 | 从商店购买精灵球 |
| 3004 | 400 | 捕捉会话已结束 | 重新发起捕捉 |
| 3005 | 400 | 精灵已逃走 | 寻找新的精灵 |
| 3006 | 400 | 精灵仓库已满 | 释放或转移精灵 |

## 错误码使用指南

### 客户端处理建议

1. **根据 code 判断业务逻辑**：不要依赖 HTTP 状态码
2. **显示 message**：直接展示给用户（已本地化）
3. **使用 traceId 报错**：提交工单时附带 traceId
4. **重试策略**：
   - 429 错误：等待 1 分钟后重试
   - 502/500 错误：指数退避重试（1s, 2s, 4s）

### 服务端日志

每个错误都会记录日志，包含：
- traceId：全链路追踪
- userId：用户 ID
- requestId：请求 ID
- stack：错误堆栈（仅开发环境）
