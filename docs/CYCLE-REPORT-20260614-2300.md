## 1. 新增需求

### REQ-00215: API 请求签名验证与重放攻击防护系统

**类别**: 安全加固  
**优先级**: P1  
**状态**: new

**背景**: 当前系统缺少对 API 请求的完整性和真实性验证，存在请求篡改和重放攻击风险。

**核心功能**:

- 时间戳验证 (防止过期请求)
- Nonce 防重放系统 (Redis)
- 多级安全策略 (L1-L4)

**涉及服务**: gateway、所有微服务、backend/shared、game-client
