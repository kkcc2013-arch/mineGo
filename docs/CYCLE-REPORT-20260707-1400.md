### REQ-00485: 用户数据批量导出安全管控与审计系统

**类别**: 反作弊/安全加固  
**优先级**: P1  
**状态**: done ✅

**实现概要**:

#### 新增文件（6个，总计 ~45KB）

| 文件 | 功能 | 代码量 |
|------|------|--------|
| `exportRateLimiter.js` | 多层限流中间件 | 5855 bytes |
| `exportApprovalWorkflow.js` | 批量导出审批工作流 | 8522 bytes |
| `dataMaskingEngine.js` | 敏感数据脱敏引擎 | 8480 bytes |
| `exportAnomalyDetector.js` | 导出异常检测器 | 9298 bytes |
| `secureExportService.js` | 安全导出服务整合 | 7012 bytes |
| `secureExportRoutes.js` | API 路由（11个接口） | 5730 bytes |

#### 核心功能实现

1. **多层限流机制**
   - 用户级：每月最多2次导出，30天滚动窗口
   - 管理员级：每日最多10次批量导出
   - 单次上限：最多1000用户

2. **智能审批工作流**
   - 小批量(1-100用户)：1人审批
   - 中批量(101-500用户)：2人审批
   - 大批量(501-1000用户)：3人审批
   - 事务保护，防止并发问题

3. **精细化数据脱敏**
   - 支持10种数据类型脱敏
   - 邮箱脱敏：a***b@example.com
   - 电话脱敏：显示后4位
   - GPS精度降低：精度降至100米
   - 基于角色权限控制可见性

4. **全面的异常检测**
   - 5种检测算法：
     - 快速重复导出检测
     - 管理员异常批量导出
     - 异常时段导出（凌晨2-5点）
     - 突发性导出检测
     - 历史行为基线对比
   - 风险评分：0-100分
   - 自动推荐处理措施（阻止/MFA/监控）

#### API 接口（11个）

| API | 方法 | 功能 |
|------|------|------|
| `/api/export/user` | GET | 用户导出自己的数据 |
| `/api/export/batch-request` | POST | 管理员申请批量导出 |
| `/api/export/history` | GET | 获取用户导出历史 |
| `/api/export/pending-requests` | GET | 获取待审批列表 |
| `/api/export/approve/:requestId` | POST | 审批导出请求 |
| `/api/export/reject/:requestId` | POST | 拒绝导出请求 |
| `/api/export/execute/:requestId` | POST | 执行已批准的批量导出 |
| `/api/export/anomalies` | GET | 获取最近异常 |
| `/api/export/high-risk-users` | GET | 获取高风险用户列表 |
| `/api/export/limit-status` | GET | 获取用户当前导出限制状态 |
| `/api/export/init-tables` | POST | 初始化数据库表 |
