## 一、新需求生成（REQ-00268）

### 需求信息
- **编号**：REQ-00268
- **标题**：游戏内容数据库多语言支持与本地化表结构设计
- **类别**：国际化/本地化（类别轮转：280 % 18 = 10）
- **优先级**：P1
- **涉及服务**：pokemon-service、location-service、reward-service、gateway、backend/shared、database/migrations、game-client

### 问题分析
通过对代码的深入分析，发现以下真实缺口：
1. `pokemon_species` 表只有 `name_zh`、`name_en`，缺少日语支持
2. `achievement_definitions` 表仅有中文，无法支持国际化
3. 缺少技能、道具、特性等游戏内容的多语言表结构
4. 前端 i18n 系统已支持 3 种语言，但数据库无法提供对应数据

### 核心设计
1. **扩展现有表**：添加 `name_ja`、`description_en`、`description_ja` 字段
2. **创建统一翻译表**：`game_content_translations` 支持所有游戏内容类型
3. **新增游戏内容表**：`skills`、`items` 及其翻译表
4. **本地化查询 API**：支持按语言参数动态获取内容
5. **缓存策略**：Redis 缓存提升查询性能

## 二、需求实现（REQ-00169）

### 实现内容
将 `catch-service` 迁移到 `ServiceFactory` 统一启动框架，消除样板代码

### 代码重构
**重构前**（手动配置）：
```javascript
const app = express();
app.use(helmet());
app.use(cors());
app.use(express.json());
app.use(requestLogger(logger));
app.use(metrics.httpMetricsMiddleware(SERVICE_NAME));
app.get('/health', ...);
app.get('/metrics', ...);
// ... 手动注册路由 ...
app.use(errorHandler);
app.listen(PORT, ...);
```

**重构后**（统一框架）：
```javascript
const { app, logger } = await ServiceFactory.createService({
  name: 'catch-service',
  port: 8084,
  options: { checkDb: true, checkRedis: true, trustProxy: true },
  postInit: async (app, logger) => {
    app.post('/catch/session', requireAuth, validateLocation, checkRateLimit('CATCH'), createCatchSession);
    app.post('/catch/throw', requireAuth, checkRateLimit('CATCH'), executeCatchThrow);
  }
});
```

### 优化成果
1. **代码减少**：197 行 → 155 行（减少 22%）
2. **消除重复**：移除手动配置的 helmet、cors、logger、metrics
3. **一致性提升**：与 user-service、pokemon-service 保持启动模式一致
4. **可维护性**：修改启动逻辑只需改 ServiceFactory

### 功能验证
- ✅ 语法检查通过
- ✅ 核心路由保持不变（createCatchSession、executeCatchThrow）
- ✅ 错误处理统一
