## 1️⃣ 新需求生成

### REQ-00291: API 密钥与敏感配置安全管理及自动轮换系统

**类别**: 安全加固  
**优先级**: P1  
**创建时间**: 2026-06-22 11:00

#### 背景问题
当前系统存在严重安全隐患：
- 硬编码默认值（`pmg-access-secret-change-in-prod`）
- 缺乏密钥轮换机制（JWT、API 密钥、数据库密码）
- 环境变量明文存储
- 无密钥访问审计
- 缺乏泄露检测

#### 核心功能
1. **密钥管理服务（KMS）**
   - 密钥分类：JWT_SECRET、API_KEY、DATABASE_PASSWORD、ENCRYPTION_KEY 等
   - 加密存储：AES-256-GCM
   - 版本管理：支持多版本并存

2. **自动轮换机制**
   - 按敏感等级轮换（critical 30 天、high 90 天、medium 180 天）
   - 零停机轮换策略
   - 旧版本保留 24 小时

3. **密钥泄露检测**
   - Git 预提交钩子
   - 日志脱敏中间件
   - GitHub 泄露监控

4. **应急响应机制**
   - 紧急撤销 API
   - 紧急轮换流程

5. **迁移计划**
   - 从环境变量迁移到 KMS
   - 更新所有使用密钥的服务

#### 验收标准
- [ ] 所有密钥存储在 KMS
- [ ] 密钥加密存储
- [ ] 自动轮换机制生效
- [ ] 管理后台可管理密钥
- [ ] Git 钩子阻止敏感信息提交


##### 2. 六层欺诈检测器

| 检测器 | 功能 | 风险指标 |
|--------|------|---------|
| ValueDisparityDetector | 价值不对等检测 | ratio < 0.1 或 > 10 |
| AccountAnomalyDetector | 账号异常检测 | 新账号、高频交易、单向转移 |
| DeviceFingerprintDetector | 设备指纹检测 | 相同设备、设备重叠 |
| BehavioralPatternDetector | 行为模式检测 | 突然高价值获取、密码重置后交易 |
| NetworkAnomalyDetector | 网络异常检测 | 相同 IP、VPN/代理、地理距离异常 |
| GroupDetectionDetector | 团伙检测 | 密集交易网络、单向流动、账号聚集 |

##### 3. 交易确认与保护
- **冷却期机制**: 
  - medium: 5 分钟
  - high: 30 分钟
  - critical: 24 小时

- **交易回滚**: 
  - 24 小时回滚窗口
  - 原子性回滚操作
  - 审计日志记录

- **团伙检测**: 
  - 交易网络图谱构建（BFS）
  - 异常聚集检测
  - 嫌疑分数计算

##### 4. API 接口
- `POST /trades/request`: 集成欺诈检测，返回风险分析
- `POST /trades/:id/rollback`: 交易回滚
- `GET /trades/analytics/report`: 异常交易报表（管理员）
- `GET /trades/fraud/rings`: 欺诈团伙检测（管理员）

##### 5. 数据库设计
- `trade_fraud_analysis`: 欺诈分析结果
- `trade_value_warnings`: 价值警告
- `trade_audit_log`: 审计日志
- `trade_rollbacks`: 回滚记录
- `user_security_events`: 安全事件
- `trade_network_cache`: 网络分析缓存
