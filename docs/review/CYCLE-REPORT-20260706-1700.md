## 任务 1：生成新需求 REQ-00465

### 需求信息
- **编号**: REQ-00465
- **标题**: API响应分页标准化与性能优化系统
- **类别**: API 设计规范（轮转编号 463 % 18 = 7）
- **优先级**: P1
- **状态**: new
- **涉及服务**: backend/shared/pagination、gateway、所有微服务

### 需求概述
当前多个微服务的分页响应存在不一致问题（参数命名不统一、响应格式不一致、大 offset 性能差）。本需求构建统一的 API 分页标准化系统，包括：

1. **统一分页参数**: 标准化 `page/pageSize/cursor/direction` 参数命名
2. **统一响应格式**: 所有分页响应包含相同的 `meta.pagination` 元数据
3. **智能分页策略**: 自动选择 offset-based 或 cursor-based 分页
4. **性能优化**: 游标分页避免大 offset 性能问题
5. **分页中间件**: 统一的参数解析和响应包装

### 关键设计
- `PaginationMiddleware.js`: 分页参数解析和响应包装
- `CursorPaginator.js`: 游标分页实现，避免 offset 性能问题
- `PaginationStrategySelector.js`: 智能策略选择器
