## 实现检查清单

### 核心模块
- [x] **SlowQueryCollector.js** - 慢查询收集器
  - 支持 pg_stat_statements 扩展
  - 慢查询阈值可配置（默认 500ms）
  - 查询标准化（移除常量，保留结构）
  - 缓存命中率计算
  - 查询严重程度分级（critical/high/medium/low）
  - 获取查询执行计划
  - 获取表统计信息和现有索引

- [x] **IndexRecommender.js** - 索引建议生成器
  - QueryPatternAnalyzer: 解析查询模式（SELECT/INSERT/UPDATE/DELETE）
  - ColumnImportanceAnalyzer: 分析列重要性
  - 单列索引建议生成
  - 复合索引建议生成（最多 4 列）
  - 部分索引建议生成
  - 优先级计算（基于执行时间和基数）
  - 建议去重

- [x] **IndexHealthChecker.js** - 索引健康检查器
  - 检测未使用索引
  - 检测重复索引
  - 检测碎片化索引（碎片率 > 30%）
  - 检测过大索引（> 表大小 30%）
  - 生成综合建议

- [x] **IndexOptimizationExecutor.js** - 索引优化执行器
  - 执行窗口检查（默认凌晨 2-6 点）
  - 数据库负载检查（连接数、活跃查询、查询时长）
  - Dry-run 模式支持
  - 批量执行优化
  - 执行日志记录
  - Prometheus 指标导出
  - Webhook 通知

- [x] **IndexOptimizerManager.js** - 索引优化管理器
  - 整合所有模块
  - 慢查询事件监听
  - 健康检查触发
  - 完整优化流程
  - 状态摘要查询
  - Prometheus 指标更新

- [x] **indexOptimizationJob.js** - 定时任务
  - 单次执行模式
  - 定时调度模式
  - 报告生成

### 技术实现亮点

1. **安全的索引操作**
   - 所有索引操作使用 `CONCURRENTLY`，避免锁表
   - 执行前检查数据库负载
   - 限制执行时间窗口（默认凌晨 2-6 点）

2. **智能建议生成**
   - 基于查询模式分析（SELECT/WHERE/JOIN/ORDER BY）
   - 考虑列基数（cardinality）和相关性
   - 自动生成单列、复合、部分索引建议
   - 按优先级排序建议

3. **完善的健康检查**
   - 检测未使用、重复、碎片化、过大索引
   - 生成 DROP/REINDEX/REVIEW 建议
   - 提供安全的执行 SQL

4. **可观测性**
   - Prometheus 指标导出
   - 执行日志记录
   - Webhook 通知
   - 状态摘要 API
