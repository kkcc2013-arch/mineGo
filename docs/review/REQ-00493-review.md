## 部署建议

1. **环境准备**
   - 确保 Kubernetes 集群版本 >= 1.28
   - 确保 PostgreSQL 配置了主从复制
   - 确保 Redis 配置了 Sentinel 或 Cluster
   - 确保 Kafka 集群可用

2. **权限配置**
   - 确保 ServiceAccount 有足够的 RBAC 权限
   - 确保可以访问数据库和 Redis 的管理端点

3. **监控集成**
   - 确保 Prometheus 可以抓取 `/metrics` 端点
   - 配置告警规则（MTTR 超过阈值时告警）

4. **演练计划**
   - 建议在业务低峰期执行演练（如凌晨 2-4 点）
   - 建议先从低风险场景开始（Pod 崩溃）
   - 逐步增加演练频率和复杂度

## 潜在风险

1. **生产环境影响** ⚠️
   - 演练会实际注入故障，可能影响用户体验
   - 建议在预生产环境先演练

2. **数据丢失风险** ⚠️
   - 数据库故障注入可能导致数据丢失
   - 建议确保有完整的备份恢复流程

3. **误操作风险** ⚠️
   - 手动触发演练可能误操作
   - 建议增加二次确认机制
