## 4. 性能考量

### 4.1 匹配优化

- ELO 差距随等待时间扩大（200 → 500）
- 最小等待 3 秒，最大等待 60 秒
- 等待时间加权，优先匹配等待更久的玩家

### 4.2 战斗优化

- WebSocket 实时同步，延迟 < 100ms
- 回合行动并行处理
- 战斗日志批量存储

## 6. 已知限制

1. **WebSocket 集成**: 需要在 gateway 层集成 WebSocket 服务器
2. **前端组件**: 需要实现 PVP 战斗 UI
3. **排位奖励发放**: 需要定时任务处理赛季结算
4. **观战功能**: 当前版本未实现观战

## 7. 改进建议

1. **匹配优化**: 可以增加地区匹配，减少网络延迟
2. **防作弊**: 增加行动频率检测，防止脚本操作
3. **回放压缩**: 大型回放可以使用 gzip 压缩存储
4. **断线重连**: 增加断线重连机制
