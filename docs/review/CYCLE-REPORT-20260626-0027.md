## 任务 1: 生成新需求

### REQ-00329: API 端点命名规范自动校验与文档同步系统

**类别**: API 设计规范  
**优先级**: P2  
**状态**: new  

#### 需求背景
mineGo 项目已有 9 个微服务，暴露了 200+ 个 API 端点，但存在以下问题：
- 命名规范不统一（kebab-case、camelCase、snake_case 混用）
- HTTP 方法误用（GET 修改数据、POST 获取资源）
- 文档与代码不同步
- 缺少自动化校验工具

#### 解决方案
1. **API Linter 工具**：自动扫描路由定义，检查命名规范
2. **CI/CD 集成**：PR 检查中自动运行 Linter，阻止不合规代码合并
3. **文档自动生成**：从代码注释和 JSDoc 自动生成 OpenAPI 文档
4. **文档同步验证**：CI 中验证 OpenAPI 文档与实际代码一致性

#### 预期收益
- API 命名一致性提升至 95%+
- 文档与代码同步率提升至 98%+
- 代码审查效率提升 20%
- 新成员上手时间减少 30%

**文件**: `docs/requirements/REQ-00329-api-endpoint-naming-convention-linter-and-doc-sync.md`
