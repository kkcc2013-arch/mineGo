## 一、需求生成

### REQ-00337：精灵详情语音描述与盲人友好信息系统

- **编号**：REQ-00337
- **类别**：无障碍(a11y)
- **优先级**：P1
- **状态**：new

#### 需求背景

mineGo 作为 AR 精灵捕捉游戏，需要为视觉障碍玩家提供完整的无障碍支持。当前 `frontend/game-client/src/accessibility/` 目录已实现键盘导航、屏幕阅读器公告等基础功能，但精灵详情页面的语音描述仍不完善：

1. 精灵详情缺乏结构化语音描述
2. 捕捉场景缺少音频反馈
3. 精灵属性数据未适配语音输出
4. 缺少精灵语音档案库

#### 需求内容

1. **精灵详情语音描述生成器**（PokemonVoiceDescription.js）
   - 生成完整语音描述（摘要、外观、属性、技能、进化路径）
   - 支持多语言本地化

2. **捕捉场景音频反馈系统**（CatchAudioFeedback.js）
   - 距离提示音（距离越近音调越高）
   - 捕捉成功/失败音效
   - 精灵移动提醒

3. **后端精灵语音档案 API**
   - `/api/pokemon/:id/voice-description` - 单个精灵语音描述
   - `/api/pokemon/voice-descriptions` - 批量获取

4. **WCAG 2.1 AA 合规**
   - 非文本内容替代文本
   - 动态内容音频描述

## 二、需求实现

### REQ-00333：游戏内智能 Toast 通知系统与用户反馈优化

- **编号**：REQ-00333
- **类别**：前端体验
- **优先级**：P1
- **状态**：done ✅

#### 实现内容

##### 1. ToastManager.js（约 500 行）

**核心功能**：
- ✅ 多优先级队列管理（5 级：critical/error/warning/success/info）
- ✅ 可撤销操作机制（10秒倒计时）
- ✅ 消息历史持久化（localStorage，最多 100 条）
- ✅ 用户行为分析统计（shown/clicked/dismissed/undone）
- ✅ 与 ErrorHandler 集成（自动识别可重试错误）
- ✅ 历史记录面板（Ctrl+H 打开）

**API 示例**：
```javascript
// 基础用法
toastManager.success('捕捉成功！');
toastManager.error('网络连接失败');

// 可撤销操作
toastManager.showWithUndo('精灵已删除', () => { /* 恢复精灵 */ });

// 错误重试
toastManager.showErrorWithRetry(error, () => retryOperation());

// 自定义按钮
toastManager.show({
  type: 'info',
  title: '新活动',
  message: '限时捕捉活动已开启',
  actions: [
    { label: '查看详情', onClick: () => goToEvent(), primary: true }
  ]
});
```

##### 2. ToastManager.css（约 300 行）

**样式特性**：
- ✅ 6 种位置变体（top-right/top-left/bottom-right/bottom-left/top-center/bottom-center）
- ✅ 5 种类型样式（critical/error/warning/success/info）
- ✅ 高对比度模式支持（prefers-contrast: high）
- ✅ 减少动画支持（prefers-reduced-motion: reduce）
- ✅ 响应式布局（移动端自适应）
- ✅ 无障碍焦点样式
