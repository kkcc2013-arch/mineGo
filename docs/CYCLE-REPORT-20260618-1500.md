- **类别**：安全加固
- **优先级**：P1
- **状态**：done

### 实现概述
实现基于用户信誉度和系统负载的智能限流系统，动态调整 API 请求配额，支持临时配额提升。

### 新增文件
| 文件路径 | 说明 |
|----------|------|
| `backend/shared/UserReputationScore.js` | 用户信誉度评分系统（6 维度） |
| `backend/shared/IntelligentRateLimiter.js` | 智能限流中间件（滑动窗口算法） |
| `gateway/src/middleware/intelligentRateLimit.js` | Gateway 限流中间件 |
| `gateway/src/routes/admin/rateLimitAdmin.js` | 限流管理接口（6 个 API） |
| `database/migrations/20260618_150000__intelligent_rate_limit_system.sql` | 数据库迁移 |
| `backend/tests/unit/UserReputationScore.test.js` | 信誉度评分测试 |
| `backend/tests/unit/IntelligentRateLimiter.test.js` | 限流器测试 |
| `docs/review/REQ-00234-review.md` | 审核报告 |

### 核心功能
1. **用户信誉度评分**：6 维度评分（账号年龄、活跃一致性、违规历史、支付可靠性、社交信任、游戏行为规范性）
2. **信誉等级划分**：5 级（NEW/BRONZE/SILVER/GOLD/PLATINUM），配额倍数 0.5-1.5
3. **智能限流**：滑动窗口算法，Redis 有序集合实现
4. **动态配额计算**：基础配额 × 信誉倍数 × 系统负载倍数 × 临时提升倍数
5. **系统负载监控**：CPU、内存使用率监控，自动降级配额
6. **临时配额提升**：支持活动期间临时提升，最长 24 小时
7. **管理接口**：6 个管理接口（配额查询、提升、重置、信誉查询、调整、统计）
